# 异常协议报文检测与防护系统 · 详细设计文档

**版本** 3.0　**日期** 2026-07-20　**定位** 通信设备(路由器/交换机)控制面协议栈内嵌部署　**适用协议** OSPF / IS-IS / BGP,及链路层报文处理模块(LDM)等

---

## 0. 阅读指引:这套系统是什么、不是什么

先把立场讲清楚,因为它决定了后面每一个设计取舍。

**它是一套"异常报文事后归因 + 免疫记忆"系统,辅以两道事前的先天防线;它不是一套"根因枚举 + 事前约束"系统。** 现网协议模块的崩溃、路由误删、状态机卡死,绝大多数源于**未知的实现缺陷**。缺陷无穷无尽、逐个不同,试图为每一类缺陷预置一条检查永远追不上,也不可维护。本系统把协议实现当**黑箱**:平时不理解它,等它出问题之后,靠**关联而非理解**,把"是哪个报文把它搞坏的"快速揪出来,记下报文特征,复发即拦。

系统内只有两处"枚举",枚举的都不是缺陷:

- **规范校验**枚举的是协议规范本身(RFC 的编码约束与明文强制条款)——数量有限,一次建立,不随缺陷增长。
- **能力校验**枚举的是**本机实现自己的能力边界**(支持哪些地址族、哪些下一跳编码、哪些特性组合)——实现团队对此最清楚,同样有限,随版本演进而非随缺陷增长。合法、但落在本机能力范围之外的输入,一律不放进未经设计的代码路径,而是执行**定义化降级**(按规范允许的方式明确地不用它,并告警)。

**它检测的是"设备可见地坏掉"的形态,不是根因。** 根因不可枚举,但"设备坏掉"的可见形态少而稳定——崩溃、状态机卡死、批量删路由、报文风暴。装一个"批量删路由"探测器,能抓住所有以批量删路由形式发作的缺陷,不管底层根因是什么。这一断言的边界同样写明:静默黑洞、缓慢劣化等"不响亮"的形态在核心探测之外,由补充审计弥补并明示范围(§4.1)。

**持久化免疫库的首要价值,是打断死循环。** 死循环有两种形态:**重收型**(崩溃 → 重启 → 从邻居重新学到毒化路由 → 再次崩溃)靠入口拦截打断;**回放型**(毒化对象从 checkpoint/GR 状态回放中复活,无需重收任何报文)靠"免疫库先于回放加载 + 驻留隔离扫描"打断,其兜底是保护性冷启动(§8.6)。要说清楚的是:冷启动确定性打断的是**回放臂**;若毒物在网络中仍有来源,冷启动后循环会转为重收型——但这恰好把"没有报文可查"的回放循环转换成了"每一轮都有报文证据"的重收循环,归因由此获得样本,拦截由入口承接。免疫库必须在协议模块恢复与状态回放**之前**加载生效,才能真正止血。

**诚实的能力边界(不粉饰):**

- 对需要**学习**的特征,首个受害者仍会中招——这是签名式防护的本质;但落在规范校验与能力校验覆盖内的异常(如超出本机能力的下一跳编码)可以**事前**拦住,首台也不中招。
- 纯竞态/时序缺陷提炼不出报文特征,只能靠自愈兜底。
- 输入完全处于规范与本机能力范围之内、由纯内部缺陷引发的故障,系统会正确地判定"这不是异常报文问题",转自愈而非拦截。
- 学习特征存在逃逸空间(最小匹配条件的代价,§7.2 缓解不消除)。
- 免疫窗口内,哨兵设备为保数据库一致性仍会向邻居泛洪毒化对象(§8.5,显式决策而非掩盖)。
- 看门狗硬复位与掉电场景下,证据按平台能力降级或丢失(§5.1)。
- 免疫库双份完整性均失败时设备以空库放行运行,该窗口内重收型循环没有入口拦截,依赖网管重新下发恢复(§7.4、§13.4)。
- 一致性守护的免环保证以"存在替代路径"为前提,割点场景守护失效并转人工(§8.4)。

## 1. 背景与设计目标

### 1.1 问题背景

协议模块因收到异常报文而异常重启、误删路由、内部状态机卡死,导致业务长时间中断且无法自愈,并可能使故障扩散全网。本设计以以下真实/构造案例为锚点驱动:

| 编号 | 案例 | 本质 |
|---|---|---|
| C1 | OSPF 收到 LSID=0.0.0.0 的 Network-LSA,域内 SPF 计算次数达特殊值时触发缺陷,误删域内路由 | 报文中的异常值触发实现缺陷 |
| C2 | 对端设备(友商实现,支持 EVPN 路由携带 IPv6 下一跳)经 IPv4 BGP 会话向本机通告带 IPv6 下一跳的 EVPN 路由——按 RFC 7432/4760,EVPN 下一跳的 v4/v6 由 MP_REACH_NLRI 的 Next Hop 长度判别,**不经能力协商**,协商机制无法拒绝。本厂商实现不支持该组合(本厂商设备经 IPv4 会话不会发送 IPv6 下一跳的 EVPN 路由,接收处理路径也未按此设计):主备倒换时,EVPN 备升主向 RM_IPv4 平滑订阅下一跳,按 IPv4 长度截断 IPv6 下一跳发起订阅,返回结果不匹配,升主状态机卡在下一跳订阅态,倒换流程无法继续,备板冗余失效 | **合法、但超出本机能力范围的输入** + 内部截断缺陷,仅倒换触发 |
| C3 | LDM 解析目的为本机地址的 IPv6 IPsec 报文,命中管理面防护策略进入丢弃处理,丢弃后误释放整个报文内存,解析下一个报文时访问已释放内存,设备异常复位 | 合法报文命中丢弃路径,代码错误释放,下一个报文崩溃 |
| C4 | IS-IS 某设备时钟补偿变量内存跳变,协议计时器快 100 倍,始发 Purge;全网泛洪、反复删路由,IS-IS 路由全网震荡 | 本机内存损坏,合法 Purge 毒化全网(输入完全在规范与能力范围内) |

四个案例代表四类输入面:C1 是**值异常**(学习轨道捕获),C2 是**能力失配**(能力校验捕获),C3 是**处置等价类**(对照统计 + 处置记录捕获),C4 的输入完全正常(行为处置遏制,不出报文特征)。

#### 1.1.1 范围划界

**范围外:** BGP 短期海量 UPDATE 导致重启、BGP 反复震荡导致 CPU 冲高。这两类是资源耗尽问题,靠水位/反压/max-prefix 的优雅降级解决,机制成熟且与本系统的报文归因主线正交,单独治理。

与本系统速率探测(§4.2(4))的划界判据:

- **归本系统:单对象病理性重复。** 判据 = 对象集中度 σ(窗口内出现次数最多的单个对象占事件比)≥ θ_c,且速率超域基线一个数量级,且**无相称的会话/接口/策略事件**。典型:同一 LSP 反复 purge+重生、同一前缀高频震荡而其余对象平稳。
- **归资源治理轨(范围外):大面积合法变动。** 对象分散、伴随相称原因(会话建立/断开、策略批量变更、整表收敛)。
- **首期配置:** IGP LSP/purge 类开启检测与处置;BGP 前缀级仅观察告警、不自动处置(阈值见 §12)。

### 1.2 设计目标

1. **目标 A(在线防线):** 在报文入口识别并处理三类可判定异常——**规范违例**(违反 RFC 编码约束或明文强制条款)、**能力失配**(合法但超出本机实现支持范围,如 C2)、**已学习的异常特征命中**。默认处置是非破坏性的;能力失配走定义化降级(按规范允许的方式明确不使用该输入并告警),不把这类输入放进未经设计的代码路径。直接丢弃并非主要手段——部分异常报文直接拦截会导致邻居断连、路由不同步。
2. **目标 B(通用自愈,与 A、C 并列的主目标):** 异常事件发生后,识别发作形态、止损、自愈、阻止扩散。覆盖崩溃、卡死、批量误删、风暴。四个锚点案例全部由本目标能力兜住,且核心机制在首期交付——这是本系统最大的普适价值,配有相称的度量与资源(§15)。
3. **目标 C(免疫记忆):** 持久化免疫条目,**打断"故障 → 重启 → 毒化状态复活/重收 → 复发"的死循环**,并使"一台设备当哨兵、经人工确认后全网受控免疫"成为可能。免疫条目分三类:学习到的值特征(C1 型)、能力校验补充项(C2 型,事后归因补入支持矩阵)、处置等价类项(C3 型)。

### 1.3 设计原则(核心立场)

- **黑箱归因,不枚举根因。** 不为每个缺陷预置检查;缺陷未知,靠事后关联定位。仅有的两处枚举——规范与本机能力边界——都是有限、一次性建立、不随缺陷增长的集合。
- **能力边界显式化。** 本机支持什么,由实现团队以支持矩阵一次性声明并随版本维护;超出矩阵的合法输入一律定义化降级。"不用它"永远比"误用它"安全——被截断、被误解析的路由比被明确弃用的路由危害大得多。
- **检测发作形态,而非根因。** 核心探测覆盖崩溃/卡死/批量删/风暴;覆盖边界明示(§4.1)。
- **不重放。** 活体重放开销过大且无法还原计时器、内存布局、引用计数、竞态时序;归因改用溯源标签、驻留态/回放态扫描、跨事件对照统计与排除验证。
- **不复制路由数据。** 不物化第二份 RIB/LSDB。唯一与对象数量相关的常驻结构是**定容量**的溯源摘要仓,设硬上限并如实入表(§2.5.1)。
- **稳态成本有预算。** 稳态不做任何 O(N) 扫描类动作;每报文/每对象的增量成本逐项核算并设上限(§11);归因等重活只在异常触发后的慢路径运行。
- **处置匹配发作路径。** 同一特征命中,处置随缺陷发作在计算侧/接收侧不同而不同(§8.3)——"隔离出计算"只对计算侧缺陷打断循环。
- **任何改变本机路由计算输入的处置,统一过一致性守护门。** 单机偏离域内共同的计算输入会制造持久转发环,这条不变式对隔离、骨架入库、计算暂缓、泛洪抑制一体适用(§8.4),不允许任何一条处置路径绕开。
- **防护动作自身纳入风险治理。** 熔断、引流、自隔离、保护性冷启动、自动挂起/回滚等动作都可能被误触发或被攻击者诱导,一律设佐证前置、并发保护、频率上限与可回滚(§8.6/§13)。
- **底层原语显式设计。** 复位可存活内存、跨进程发布与回收、时间基准、RIB 摘要一致性等,一律不"假设存在",逐一给出设计与核算。
- **默认非破坏 + 人在环。** 系统产出"候选 + 证据档位 + 证据档案"交人晋级;自动化边界由证据档位机械限定(§6.3)。
- **本机学习本机生效,不自动向邻居扩散阻断规则。** 跨设备走现有可信配置分发通道 + 签名 + 灰度 + 可回滚。

### 1.4 术语

- **PDU IR(协议规范化字段树):** 报文在解析瞬间的规范化字段视图,复用协议自身解析结果映射而来,仅存在于处理的那几微秒,不建并行数据库。
- **支持矩阵 / 能力失配 / 定义化降级:** 本机实现声明的"能正确处理的输入组合"清单(按地址族 × 下一跳编码 × 会话类型 × 特性开关等维度);落在矩阵外的合法输入称能力失配;对其执行的、协议规范允许的明确弃用处置(按撤销处理、忽略不参与选路、属性丢弃)称定义化降级。
- **溯源号(input_id):** 挂在驻留协议对象上的 8B 引用 {环号, 代次, 环内序号},指回创建/最后修改该对象的接入事件。
- **事件时间轴:** 黑匣子中严格按时间老化的纯环形日志(接入 + 状态变更),永不钉住。
- **溯源摘要仓:** 按溯源号索引、定长槽位、有硬容量上限的关键字段摘要存储;可容忍失引。
- **复位域 / 证据存活等级:** 按复位类型(软复位/硬看门狗/掉电)划分的证据可存活程度(§5.1)。
- **回放循环 vs 重收循环:** 死循环的两种形态——毒化状态经 checkpoint/GR 回放复活 vs 经邻居重新学习。
- **绊线:** 通用异常触发器,一条覆盖一整类发作形态。
- **发作路径:** 缺陷发作所在的代码侧别——计算侧(SPF/选路)、接收/安装侧(解析/入库)、状态机/内部侧。
- **排除验证:** 把头号候选从输入中排除后重算一次,检验"排除它则症状消失"。有两种执行体:生产实例的受保护试算(首选)与生产同码影子(§6.4)。
- **独立参考计算器:** 与生产代码无关的极简正确实现,仅用于裁决"本机是否算错"(§8.6),不用于排除验证。
- **证据档位(E0–E3):** 归因结论的离散证据强度分级,决定允许的自动化程度(§6.3)。
- **骨架入库:** 仅解析对象头部用于数据库身份、同步对账与确认,原始字节参与泛洪/重传,不做体解析、不安装内容(§8.3)。
- **线缆视图 / 计算视图:** 协议数据库与泛洪按标准处理(线缆视图)与该状态被应用到路由计算(计算视图)的分离(§8.6)。
- **一致性守护:** 所有改变本机路由计算输入的处置的强制前置门(§8.4)。
- **让位窗口:** 循环模式下,状态回放完成后、首次全量计算前,为回放态扫描预留的短暂窗口(§6.1)。
- **特征 / 驻留态 / 黑匣子 / 观察态·执行态:** 免疫条目及其匹配条件;已进入协议数据库长期存在的状态;复位可存活的证据区;免疫条目"只记录不干预"与"实际执行处置"两种生效状态。

### 1.5 与出厂前加固的关系

本系统不替代协议模糊测试、带内存检测工具的构建与倒换场景系统测试——那些手段在出厂前**消灭**缺陷,本系统是**漏网缺陷的现网遏制**,二者互补。诚实地说:C3 类内存释放后使用缺陷恰是模糊测试 + 地址检测工具廉价可捕的类型,该投入应持续;C2(倒换时序 + 跨厂商能力差异)与 C4(运行期内存损坏)则是出厂测试的固有弱区,是现网机制的相对优势区。为让这一判断可被数据检验:§14 的回溯研究对每个历史缺陷同时标注"出厂加固能否捕获";若可捕占比很高,应向立项方建议投资再平衡。本系统进程内框架自身也按同样标准加固(§2.3)。

## 2. 总体架构

### 2.1 双轨结构

系统分两条轨道。**在线轨**是报文处理流水线,每个报文都走,稳态只跑有预算的廉价动作;**离线轨**是免疫闭环,只在出事时才被触发。

```
                          ┌──────────────── 在线轨(每报文) ────────────────┐
   报文入口 ─► 归一化IR ─► 规范校验+能力校验 ─► 免疫库匹配 ─► 协议状态机 / LSDB / RIB
   (record_input)          (先天防线 §3)        (免疫记忆)     (report_event 上报状态变更
        │                                          ▲            record_verdict 回填处置结果)
        │ 写接入日志                               │ 加载/命中          │
        ▼                                          │                   ▼
   ┌────────────── 黑匣子(按复位域分级存活 §5.1)───────────────────────────┐
   │  事件时间轴(接入+状态变更,纯环形) · 溯源摘要仓(定容量) —— 统一时间轴  │
   └────────────────────────────────────────────────────────────────────────┘
        ▲                                                            │
        │ 证据                                                       │ 触发
        │                          ┌──────────── 离线轨(仅出事时) ──┘
        │                          ▼
   健康监测(触发器 §4) ─► 归因定位(§6) ─► 免疫条目生成(§7) ─► 持久化免疫库 ─► (人工审核后)分发(§9)
   崩溃/卡死/批量删/风暴    漏斗+排除验证    泛化+证据档位      掉电不丢
                                   │
                                   └──► 低置信/输入正常 → 自愈 + 上报(§8),不出条目
```

在线轨对应目标 A;健康监测 + 分级处置与自愈对应目标 B;黑匣子 + 归因 + 条目生成 + 持久化对应目标 C。

### 2.2 五级处理流程

离线轨的核心是五级流水线,全程把缺陷当未知:

1. **异常检测(触发器)** —— 通用绊线识别"设备坏掉"的形态。
2. **黑匣子(证据留存)** —— 按复位域分级存活,报文与状态变更同一时间轴。
3. **归因定位** —— 漏斗式从"记的所有东西"收敛到 1–3 个候选 + 理由 + 发作路径,并经排除验证把"统计罕见"与"因果致病"分开。
4. **免疫条目生成与持久化** —— 判别字段泛化成匹配条件,条件宽度随处置安全性联动;掉电不丢。
5. **分级处置与自愈** —— 非破坏处置为默认,处置匹配发作路径;止损、隔离、自愈,阻止扩散。

### 2.3 部署形态与组件

- **捕获面(协议进程内):** 钩子(§2.4)在报文热路径上由协议 inline 调用,必须在进程内——热路径不能引入进程间通信。协议团队实现 IR 映射(复用自身解析结果)、状态上报点与处置执行(工作量按 §2.4 矩阵计入 §14)。
  **进程内框架代码纪律:** 框架在协议进程内的全部热路径代码(IR 映射消费、规范/能力校验、免疫库匹配、环写)遵守——不做动态内存分配、**不做任何内存释放**、所有字段访问带边界校验、单对象执行时间有硬上界、任何内部错误只有两种降级:安全放行或按既有策略丢弃,**绝不允许崩溃传播**;纳入持续模糊测试与内存/边界检测构建门禁;提供一键旁路(整层直通)。
  **故障隔离边界的诚实声明:** 独立分析进程隔离的是分析面;进程内框架代码不被进程边界隔离,其可靠性由上述纪律与测试保障——不宣称"框架缺陷不影响协议"。
- **分析面 / PAD 服务(控制面独立进程):** 承载触发器汇聚、归因引擎、免疫库管理、字段分布模型;独立进程是为故障隔离——协议崩溃后,一个全新的 PAD 仍能读回证据做事后归因。由**监护进程**管理:自动重启、CPU/内存上限、读者进度槽位清理(§2.5.3)、告警。失效模型见 §2.5.4。
- **持久化存储:** Flash **仅承载免疫库、事件证据档案、标定数据与稳态基线快照(均为慢路径写)**;热路径日志一律走复位保留内存(§5.1)。不使用 NVRAM/Flash 作热路径日志载体:字节可写 NVRAM 在主控板容量不现实,Flash 受擦写带宽与磨损寿命双重限制,与热路径速率相差数量级。
- **线卡执行插件:** 纯头部可判的免疫条目下压 NP/ACL/CoPP;覆盖面如实收窄见 §11;规则版本协商与命中回流见 §2.6。
- **网管侧分析器:** 跨设备模型汇聚与基线交叉校验、置信度标定、候选汇总、人工审核与灰度分发、加速审批通道(§8.5);同时保有全网免疫库副本,作为设备本地库损坏后的恢复源(§7.4)。

### 2.4 协议模块钩子 API

```c
// 能力注册:协议启动时声明各对象类支持的处置级别,以及本协议的输入支持矩阵。
// 不支持的处置级别被免疫条目命中时,框架自动降为"观察 + 告警"并反馈 PAD。
void          register_caps(protocol_id_t p, const disposition_caps_t *caps,
                            const support_matrix_t *sm);

// 报文/对象入口:规范校验 + 能力校验 + 免疫库匹配,返回处置
disposition_t ingress_check(const pdu_ir_t *ir);

// 写接入日志(事件时间轴),返回溯源号 = {环号, 代次, 环内序号}
input_id_t    record_input(const pdu_t *pdu, const meta_t *meta);

// 回填该接入事件的最终处置结果(接受 / 按策略丢弃+原因码 / 出错丢弃 / ...)。
// 接入条目在报文处理结束时一次性发布,处置结果内联,不产生第二次环写;
// 这是"处置等价性"证明(§6.6)的证据来源。
void          record_verdict(input_id_t id, verdict_t v);

// 给驻留对象打溯源标签,同时把关键字段摘要晋级进溯源摘要仓(§5.3)
void          tag_object(object_t *obj, input_id_t id, const key_digest_t *kd);

// 上报状态变更:LSDB/RIB 增删、状态机迁移,条目内联其成因的溯源号;
// 支持批量聚合形态("邻居X 批量安装 N 条,首末序号"),收敛期不逐条
void          report_event(const state_event_t *ev);

// 处置执行结果反馈:执行失败 / 能力降级时告知框架与 PAD(计入证据档案与告警)
void          report_disposition_result(feature_id_t f, input_id_t id, dispo_result_t r);

// 免疫库查询:协议侧直读共享免疫区(§2.5.3),不依赖 PAD 存活
const feature_t *query_signature(const pdu_ir_t *ir);
```

**处置执行者矩阵。** 各处置的执行者与协议侧改造量级如实列出,工作量计入 §14 分阶段交付:

| 处置 | 执行者 | 协议侧改造量级 |
|---|---|---|
| 观察 / 记录 | 框架 | 无 |
| 入口丢弃(红线外的载荷对象) | 框架(ingress_check 返回处) | 低 |
| 定义化降级(能力失配:按撤销处理 / 忽略不选路 / 属性丢弃) | 协议(BGP 复用 RFC 7606 语义) | 低–中 |
| 净化(属性丢弃 / 字段裁剪) | 协议 | 低–中 |
| treat-as-withdraw(按撤销处理) | 协议(BGP 已有) | 低 |
| 隔离(计算侧):入库+泛洪+不参与计算 | 协议:LSDB 逐对象隔离位 + 计算遍历跳过 | **高(侵入 LSDB 与计算内核)** |
| 骨架入库(§8.3) | 协议:接收路径改造 | **高** |
| 关 AFI/SAFI / 降级 | 协议 | 中 |
| purge 计算视图暂缓 + 泛洪节奏控制 | 协议 | 中–高 |
| 路由删除熔断(冻结 FIB 删除/撤销传播) | 协议 + RM | **高** |
| CoPP/ACL 丢弃 | 线卡插件 | 中(限头部类) |
| max-metric / overload / GRACEFUL_SHUTDOWN | 协议(复用既有机制) | 低 |

### 2.5 数据获取与进程边界

分析面是独立进程,PAD 拿协议数据的方式是**协议向外推送,不是 PAD 向里拉取**。

```
  协议进程(捕获面·进程内)                    PAD 进程(分析面·独立)
  ┌──────────────────────────┐             ┌──────────────────────────┐
  │ 报文热路径 · 钩子          │  ③ 服务请求 │ 归因引擎 / 字段分布模型   │
  │ LSDB / RIB / 状态机       │ ◄─────────► │ 免疫库管理 / 标定         │
  │ 慢路径 worker 线程        │ (计算下沉)  │ (仅回传 top-K 候选)      │
  └───────────┬──────────────┘             └──────┬───────────▲───────┘
              │ ① 推送写入(热路径)                │ ② 读取    │ ①′ 异步消费
              ▼                                    ▼           │ (训练模型)
  ┌──────────────── 共享复位保留区(每写者环 + 摘要仓 + 免疫区)──────────┐
  │ 事件时间轴 · 溯源摘要仓 · 免疫库(双区) —— 协议崩溃后仍可读            │
  └──────────────────────────────────────────────────────────────────────────┘
  既有流 BMP(仅 BGP,可选富化)──────────────────────► PAD
```

| 路径 | 承载数据 | 是否落副本 | 触发时机 | 用于 |
|---|---|---|---|---|
| ① 钩子推送到共享区 | 报文指纹、状态变更、溯源摘要、处置结果 | 否(时间轴有界,非路由镜像) | 热路径 | 全部;卡死溯源(C2) |
| ①′ PAD 异步消费接入日志 | 逐条归一化字段值(消费即弃) | 否 | 稳态低优先级 | **字段分布模型的唯一训练源** |
| ② BMP(RFC 7854) | BGP Adj-RIB-In 流 | 否(流式即弃) | 稳态 | 可选增强:BGP 字段基线富化 |
| ③ 服务请求(计算下沉) | 仅 top-K 候选 + 字段 | 否(不拷贝 LSDB) | 慢路径,仅协议存活 | 驻留态扫描(C1)、回放态扫描 |

BGP-LS 不用于字段模型训练、不作算错裁决的输入:它是拓扑抽象导出(RFC 9552),不承诺对 LSA/LSP 逐字段保真,仅作旁路参考(§8.6)。

**① 钩子推送(热路径,无进程间通信)。** `record_input / tag_object / report_event` 只是往本线程独占的共享内存环写几十字节,协议不阻塞、不等 PAD。`report_event` 上报状态机卡死时把卡住对象的溯源号一并带出,PAD 经溯源摘要仓直接解析,不读协议私有内存(C2 走这条;跨板语义见 §2.6)。

**③ 服务请求:把计算下沉到数据(慢路径,仅协议存活时)。** 唯一需要读活体状态的是驻留态/回放态异常扫描。PAD 不去够协议的私有堆,而是发请求,协议在一个非阻塞 worker 线程上遍历自己的 LSDB、对每个对象套 PAD 给的**声明式打分规格**、只回传 top-K 候选 + 字段值。执行协议(锁纪律、分片、超时降级、结果信任标注)见 §6.2(B)。

**按故障态分流(独立进程的价值):** 协议**活着**时(C1 误算、C2 卡死),活体状态可得;协议**死了**时(C3 崩溃),归因由全新的 PAD 读回共享区时间轴、跨事件对照统计完成;协议**死而复生且处于回放循环**时,PAD 在让位窗口内调度回放态扫描(§6.1)。这是共享区必须复位可存活、分析面必须独立于协议进程的全部理由。

#### 2.5.1 内存模型:黑匣子双结构

"有界环形日志"与"溯源引用永不失效"是一对无法同时成立的承诺——最老的日志条目往往正是最长寿路由的创建记录,若为保引用而钉住条目,环会卡死或退化为搬移结构。本设计把两个诉求拆到两个各自守约的结构上:

- **事件时间轴(纯环形):** 接入日志 + 状态变更日志,**严格按时间老化,永不钉住**;每写者一环(§2.5.2);容量 = 窗口 × 速率封顶,收敛期最坏情形核算见 §5.5。
- **溯源摘要仓(定容量):** 按溯源号索引的定长槽位表,存"创建过驻留对象的报文"的归一化关键字段摘要(64–128B/槽)。**硬上限,缺省 32MB ≈ 26 万槽(机型可调)**;近似最久未用淘汰,打标签时刷新热度;设**免逐出位**(§5.3)保护关键槽;不维护反向引用计数。
- **失引语义:** 标签解析可能返回"摘要已淘汰"。此时溯源链**降级而非失败**:退回驻留态扫描或跨事件对照统计。关键在于:**驻留态类免疫条目的匹配条件来自异常对象自身的归一化字段,不依赖历史报文——摘要失引只降低证据链丰富度("何时从哪来"),不阻断出条目。**
- **常驻内存核算(单一权威表):**

| 结构 | 规模 | 上限 | 随路由规模增长? |
|---|---|---|---|
| 溯源标签(对象内 8B) | O(对象数) | 搭现有结构,MB 级 | 是(标签本体极小;BGP 打在共享属性集上) |
| 溯源摘要仓 | O(min(对象数, 槽容量)) | **硬上限 32MB(缺省)** | 否(封顶,超限淘汰 + 失引降级) |
| 事件时间轴 | 窗口 × 速率 | 64–256MB | 否(封顶;窗口时长核算 §5.5) |
| 字段分布模型(PAD 内) | O(字段数) | 亚 MB | 否 |
| 速率探测高频对象榜(§4.2) | 固定预算 | 2–4MB | 否 |
| 疑犯钉住配额区(§5.4) | 固定预算 | 8MB | 否 |

**为什么不建影子 RIB:** 影子 Adj-RIB-In = (指纹 + 打分字段) × 路由数 ≈ 32B × 百万路由 ≈ 32MB、× 5 百万 ≈ 160MB,随路由规模**无上限**线性增长。本设计把"摘要"这笔与对象数相关的开销做成显性定容量结构,如实入表并给出降级语义——"零额外路由内存"卖点的诚实版本:唯一与对象数相关的结构有硬上限。

#### 2.5.2 时间轴的并发与时序基准

写者的现实是多协议进程 × 每进程多线程(热路径线程、worker、看门狗)。设计如下:

- **每写者一环:** 每个写者线程经注册 API 独占一条单写单读环,环内语义严格成立。
- **环注册准入:** 环总数 × 各环容量 ≤ 时间轴总内存预算;热路径报文环容量下限 16MB(保证单环能容纳一次最坏收敛,§5.5),worker/低速环下限 1MB;协议在能力注册时声明其写者线程集(预期全系统 <32 条环);线程退出后其环转只读留证、纳入归并、按老化回收;溯源号编码 {环号:10b, 代次:10b, 环内序号:44b},环槽复用(代次回绕)须经 PAD 确认旧环已消费留证。
- **时序语义分级(不假装拥有全局因果序):**
  - **同环内:严格全序**(环内序号单调)。
  - **跨环:近似时间序**——每条目记录本地单调时钟读数(同板同步时钟源,精度微秒级以内,精度随平台声明);跨环按时戳归并,**不承诺因果序**。
  - **因果关键判断不依赖跨环排序**,依赖两条显式证据:(a) 状态变更条目**内联其成因的溯源号**(接入 → 变更的因果由数据字段直接给出,不靠时间比较);(b) **崩溃线程定位**——框架记录崩溃线程标识,该线程独占环的末尾条目就是崩溃前的最后动作(同环全序可靠)。发作路径归因(§6.7)完全建立在这两条之上。
- **条目格式与发布序:** {环内序号, 时戳, 类型, 长度, 载荷, CRC32C}。小条目(≤128B,绝大多数)以普通存储写载荷、以 release 语义发布头部;大样本条目(全量报文)以绕过缓存的写入(non-temporal)写载荷、**插入存储栅栏(sfence)**、再 release 发布头部——保证 PAD 以 acquire 读到头部时载荷必已可见,撕裂只在真实半写(崩溃)时出现并由 CRC 判弃。栅栏成本计入 §11 预算。
- **cache 纪律:** 头尾指针与数据分离缓存行;大样本走绕过缓存写,避免污染热路径二级缓存。

#### 2.5.3 免疫区发布与跨进程回收

免疫库的共享读写不能只写"版本化指针"四个字了事——跨进程没有内核那样的静止态检测,回收错误等于在 `query_signature` 热路径上亲手制造释放后使用。设计为**双区 + 读者进度槽位**:

- 共享免疫区两个存储区,各容纳完整编译后免疫集(两级索引 + 匹配条件);头部 {活动区号, 版本, 校验和}。区内**一律偏移量,无指针**(各进程映射基址无关)。
- **发布:** PAD 在非活动区完整构建 → 写校验和 → 单字原子翻转活动区号。读者检测到版本变化时重校验一次校验和,失败则回退旧区并告警——半更新态永远不可见。
- **回收(宽限期):** 每个读者线程在共享区持有独立进度槽位:进入读临界区写 {区号, 版本},退出清零;读临界区**禁止阻塞、时间上界 ≤10µs**。PAD 重建旧区前轮询全部存活读者槽位越过旧版本。
- **读者崩溃清理:** PAD 监护经进程存活检测发现读者进程死亡后清理其槽位,宽限期必然终结。
- **加载独立性:** 协议进程启动时由**协议侧框架直接**从 Flash 读免疫库文件(HMAC 验证,§7.4)构建初始区,**不等待、不依赖 PAD**;超时(缺省 500ms)或双份均校验失败 → 空库放行 + 持续告警 + 触发网管重下发恢复流程(§7.4)——处置面缺失但协议恢复不被阻塞,防护系统不成为设备收敛的单点。PAD 起来后接管后续发布。"免疫库先于回放加载"由协议侧自主完成。
- **回放前生效集限制:** 复位后立即生效执行的仅 {执行态(已过人工审批与灰度)};观察态条目仅记录。本地新学未审条目缺省观察态——"先于恢复生效"的拦截权力被限定在已审集合内。

#### 2.5.4 PAD 失效模型

| PAD 状态 | query_signature | 归因/出条目 | 系统影响 |
|---|---|---|---|
| 未启动 / 崩溃 | 正常(读现有区) | 暂停;重启后从共享区续读 | 无热路径影响;新条目暂不产出 |
| 发布中途崩溃 | 读者校验失败 → 自动回退旧区 | — | 告警;监护重启后重新发布 |
| 资源超限(O(N) 扫描/模型内存) | 正常 | 被资源上限约束或终止 | 不拖垮主控;归因延迟 |
| 与协议进程同时崩溃 | 协议重启走加载独立性路径 | 新 PAD 读保留区取证 | 设计主场景 |

#### 2.5.5 字段分布模型:数据流与训练卫生

**唯一训练路径:** 路径 ①′——PAD 以低优先级异步消费接入日志条目,在 **PAD 内**更新按协议按字段的紧凑统计(计数摘要 + 小直方图 + 均值方差),消费即弃。协议进程不做任何模型更新;网管侧只做跨设备汇聚、标定与基线交叉校验。纯 IGP 设备的 OSPF/IS-IS 基线由 ①′ 天然覆盖;BMP 仅为 BGP 可选富化。

**稳态基线快照:** PAD 每小时把归一化类出现率统计持久化一份小快照(Flash,慢路径,KB–MB 级)。它是密集崩溃循环下跨事件对照统计的对照分布来源(§6.2C)——循环期间设备大部分时间在崩溃/回放,正常窗口稀缺,对照取自循环开始前的最近稳态快照。

**训练卫生(抗自我归一化与投毒):** 驻留态扫描的判别力完全来自"当前值相对历史分布罕见",无卫生的在线自训练会被真凶或攻击者喂坏。四条纪律:

1. **双时间窗:** 长参考窗(30 天,指数衰减)+ 短近期窗(24 小时);打分对照**长窗**。
2. **隔离池延迟准入(双条件):** 异常分超 θ_q 的样本进隔离池;准入长窗须同时满足——T_admit(缺省 7 天)内未关联任何异常事件,**且复评时该值在当前长窗下的异常分已回落至 θ_q 以下**。第二个条件封死慢触发型穿透:若只有毒化对象自己在周期刷新,它的样本都被扣在池里、长窗中该值仍然罕见、复评永不回落、永不准入;只有当大量独立来源也在使用该值(说明它真是正常值)时,长窗才会自然把它变常见、复评回落、准入。
3. **新奇度 = 首见时间:** 按归一化类记录首见时刻(时间桶布隆结构),刷新不重置。评分同时使用"长窗罕见度"与"首见年龄"——真凶数小时/数天前进入,新奇信号不被它自己的周期刷新抹平。
4. **规范先验缺省启用:** 对安全关键字段(LSID、下一跳长度、保留域等)叠加协议合法值域表(与 §3 规范校验规则表同源)。此类字段的荒谬值不因本机出现频率而洗白。先验只影响**评分**,不直接触发处置,因此其收录范围可以比带处置的规范校验更积极——保留/非法地址域这类"编码合法、语义荒谬"的项**缺省列入先验**,不依赖运营商额外开启。

**网管交叉校验:** 单机分布相对全网同类设备漂移超阈 → 该机标注"基线可疑",其自动出条目通道降档为人工。

### 2.6 分布式形态:主备主控与多板

目标设备是主备主控 + 多线卡的框式形态,溯源与免疫必须跨板成立——尤其 C2:卡死发生在倒换后的新主用板,而肇事报文的接入记录在旧主用板。

- **板级归属:** 捕获面、事件时间轴、溯源摘要仓在协议进程所在主控板,**每板独立保留区、独立写、互不迁移**;PAD 运行于主用主控。倒换后新主 PAD 接管本板保留区;旧主板可达时经板间通道尽力拉取其保留区做跨板归并(各板独立环序号 + 已同步系统时钟,按时戳 + 板号归并并标注精度)。
- **溯源跨板:** NSR 对象同步时**随对象携带 {溯源号, 归一化关键字段摘要 ≤128B}**——同步的是摘要而非裸号,备板收到后写入**本板**溯源摘要仓并置免逐出位(§5.3)。倒换后新主对卡死对象做标签解析,命中的是本板摘要,溯源链闭合。带宽成本:仅对象创建/变更时随同步一次,摘要远小于对象本体。
- **免疫库:** 主备经既有配置同步通道实时同步;倒换前健康闸门校验免疫库版本一致(§8.7)。
- **线卡执行插件:** 下压规则携带代次号;线卡应答生效版本,不一致 → 告警并禁用该卡下压(退回主控软判,宁慢勿错);命中计数 + 抽样报文头周期批量回流主控,计入命中数与证据链。
- **倒换交接:** 时间轴所有权为板级,不做迁移;新主起新环续写(新代次),旧环只读留证。

## 3. 先天防线:规范校验与能力校验

在线轨入口有两道事前的门。它们与"不枚举根因"的立场不冲突:第一道枚举的是**协议规范**,第二道枚举的是**本机自己的能力边界**——两个集合都有限、一次性建立、随规范或版本演进而非随缺陷增长。"规范合法、能力范围内、但实现致病"的无穷集合,才归学习轨道。

### 3.1 规范校验

- **定义:** 仅限协议规范级、对单报文/单对象可静态机械判定的约束:编码结构约束(长度/边界/必选字段/嵌套深度)与 RFC 明文强制(MUST 级)取值约束。
- **来源与维护:** 协议团队按 RFC 一次性编写"字段合法值域表";该表**一表三用**——规范校验规则(本节)、字段分布模型的规范先验(§2.5.5)、非破坏免疫条目的匹配条件宽化依据(§7.2)。
- **处置映射:** BGP 违例直接落 RFC 7606 阶梯(净化/按撤销处理/会话级);OSPF/IS-IS 编码违例按协议既有错误处理(通常丢弃该 LSA/TLV 并计数),可疑但非强制条款违例走净化/隔离。**会话/同步类报文的违例只走协议既有错误处理,不由本系统的免疫条目处置**——受 §8.2 红线约束。规范校验的误杀风险与学习条目同样纳入 §13 治理(违例计数异常升高即告警复核)。
- **C1 为何缺省不在这道门被拦:** LSID=0.0.0.0 的 Network-LSA 属"编码合法、语义荒谬"——RFC 2328 中 Network-LSA 的 LSID 应为 DR 接口地址是语义约定,规范未提供可机械判定的强制拒收条款。带处置的规范校验只收规范明文(保守,避免自造误杀),因此缺省不含它;但它**缺省列入评分先验**(§2.5.5),使学习轨道对它高度敏感。运营商可选启用本地增强校验(把保留/非法地址域列入带处置的规则),则 C1 类在入口即被净化——两条路径并存。

### 3.2 能力校验

- **定义:** 对照本协议注册的**支持矩阵**(register_caps 随能力注册提交):按地址族/子地址族 × 下一跳编码 × 会话/通道类型 × 特性开关等维度声明"本机实现能正确处理的输入组合"。规范合法、但落在矩阵之外的输入即**能力失配**。
- **为什么必须有这道门:** 协商机制覆盖不了所有能力差异。以 EVPN 为例,下一跳是 IPv4 还是 IPv6 由 MP_REACH_NLRI 的 Next Hop 长度判别(RFC 7432/4760),**不经任何能力协商**——对端(友商实现)完全合法地经 IPv4 会话发来 IPv6 下一跳的 EVPN 路由,本机没有任何协商手段拒绝它。若本机实现不支持该组合,这条路由就会流进从未按此设计的代码路径(C2 的截断缺陷正在那里)。能力校验把这类输入在入口显式接住。
- **处置:定义化降级。** 按协议规范允许的方式明确地不使用该输入:BGP 类按撤销处理或忽略不参与选路(RFC 7606 语义),属性级问题走属性丢弃;同时记录接入日志并告警("收到超出本机能力的输入:邻居 X,组合 Y")。理由很朴素:本机反正无法正确使用它——截断后误用比明确弃用危害大得多(错误的下一跳、崩溃、卡死);明确弃用的后果(该目的地经此路径不可达)可见、可告警、可在对端或本机升级后消除。
- **维护与免疫联动:** 支持矩阵随版本发布(新版本支持了某组合 → 矩阵更新 → 校验放行);矩阵不可避免有遗漏,遗漏时的兜底是自愈路径 + 事后归因——归因发现"卡死/崩溃对象溯源到一条能力范围之外的输入"后,产出的免疫条目就是一条**能力校验补充项**(经人工确认后并入矩阵),复发与其他设备由此可拦。这类条目是确定性的(依据"本机不支持"这一可验证事实),不需要统计证据。
- **误标治理:** 把实际支持的组合误标为不支持,后果 = 该类路由被明确弃用 + 持续告警——可发现、可回滚,比截断/崩溃安全数量级。矩阵条目是配置态,变更走审计通道,与免疫库同等治理(§7.5)。

## 4. 异常检测(触发器)

### 4.1 设计立场与坏形态分类学

**"根因"无穷无尽、逐缺陷不同,不可枚举;"设备可见地坏掉"的方式少而稳定。** 触发器只盯后者——这是本系统不退化成"见一个 bug 加一条规则"的第一道保证。覆盖面如实列表:

| 发作形态 | 绊线 | 覆盖状态 |
|---|---|---|
| 崩溃 / 复位 | 核心 (1) | 覆盖(证据按复位域分级,§5.1) |
| 状态机卡死 / 流程停滞 | 核心 (2) | 覆盖 |
| 批量路由误删 / 误改(超阈) | 核心 (3) | 覆盖 |
| 单对象风暴 / 病理性重复 | 核心 (4) | 覆盖 |
| 静默转发黑洞(RIB↔FIB 不一致 / 下一跳错) | 补充:一致性审计(§4.3) | 补充覆盖 |
| 缓慢资源泄漏 / 性能劣化 | 补充:水位斜率绊线(§4.3) | **触发可及,归因大概率低置信**(真凶多已出窗)——如实列为局限(§13.4) |
| 阈下持续漂移(小批量持续误删) | 补充:累计偏差检测(§4.3,可选) | 部分覆盖 |
| 数据面硬件位错、伪装完全合法的拜占庭输入 | — | **明示范围外**(§13.4) |

**闭合声明:** 核心四类覆盖"控制面软件把状态**可见地**搞坏"的形态;静默与渐变类靠周期审计与趋势绊线补;不可观测类明示范围外。

### 4.2 四类核心触发器

**(1) 崩溃 / 复位。** 复位后 PAD 读回黑匣子做归因,可用证据量按复位域分级(§5.1)。覆盖 C3。

**(2) 状态机看门狗。** 所有状态机(BGP FSM、OSPF NSM/ISM、下一跳迭代、RIB→FIB 下发、主备批量同步)统一框架注册,强制携带 `进入时间 / 最大驻留时长 / 进展计数器`。驻留超时或进展计数不前进 = 卡死。驻留时长阈值由协议团队按状态机注册,并带场景乘子(GR/倒换期 ×5),治理见 §12。覆盖 C2 的兜底路径(主路径是 §3.2 能力校验)。

**(3) 批量路由变更 + 因果核对。** 窗口内路由增删量过阈值即触发候审。**因果一致性判据:** 若 ΔRIB 巨大,先在归并时间轴上核对是否存在**相称的原因**——不只 ΔLSDB,还包括:窗口内收到的 BGP 撤销量(≥80% ΔRIB 视为相称)、邻居/会话/接口事件、配置与策略变更事件。相称 → 合法收敛,不触发(上游合法撤销数万前缀不会被误判为"本机算错")。真触发后的动作分级:**首期 = 冻结**(不下发 FIB 删除、不传播撤销、不生成汇总/引入)+ 最高级告警;**引流(max-metric)不自动执行**,需算错裁决确认或经认证的人工确认(防被诱导引流)。解冻条件与超时见 §8.6。覆盖 C1。

**(4) 按对象的速率 / 重复模式。** 采用**有界高频对象统计**:计数摘要判"是否高频" + top-k 结构锁定具体对象,**固定预算 2–4MB,与对象总数解耦,风暴制造新对象也不增长**。精度代价如实声明:预算内保证捕获事件占比 > 1/k 的对象,长尾低频对象可能漏——可接受,风暴的定义即高占比。**BGP 前缀级:稳态仅粗探测 + 告警;精确逐前缀跟踪仅在触发后的慢路径对 top-k 对象启用。** 与范围外资源治理的划界按 §1.1.1。覆盖 C4。

### 4.3 补充绊线

- **RIB↔FIB 一致性审计:** 低频周期抽样比对(如每 10 分钟抽 1%,日级全量),不一致即触发——覆盖静默黑洞形态。
- **水位斜率绊线:** 内存/CPU/收敛时延的长窗趋势越限告警。归因预期低置信(如实声明);主要价值是在耗尽复位**之前**留出人工介入窗口,并提前抬高黑匣子保真度让后续崩溃可归因。
- **累计偏差检测:** 对"删多增少"的单向累计偏差检测阈下持续误删(可选)。

### 4.4 性能纪律

触发器的核心纪律:**监控开销正比于异常率,而非流量率。** 稳态只跑核心绊线,全部 O(1) 时间且尽量不在报文热路径——看门狗放低优先级线程周期扫,水位/速率是本来要加的计数器,因果核对只在有删除动作时在归并时间轴上比对既有计数,高频对象统计的更新是每事件常数次哈希。空间口径如实:速率探测常驻 2–4MB 固定预算,已入 §2.5.1 核算表。昂贵动作(归因、驻留扫描、算错裁决)只在绊线触发后运行。逐钩子时间成本见 §11。

## 5. 黑匣子(证据留存)

### 5.1 复位域与证据存活等级矩阵

证据能否存活取决于复位类型,必须分级设计、如实声明:

| 复位类型 | 保留 DDR 区 | 片内 SRAM 微缓冲 | 证据结论 |
|---|---|---|---|
| 软件复位(panic / 可捕获异常 / 主动重启) | **存活**(需 BSP 配合,见下) | 存活 | 全量证据 |
| 硬件看门狗复位 | **不保证**(平台相关,部分平台全芯片复位、DDR 内容不保) | **存活**(数百 KB) | 降级证据:见下 |
| 整机掉电 | 丢失 | 丢失(除非带电容/非易失 SRAM) | 首轮不可归因,如实降级;Flash 仅存免疫库与既往档案 |
| 线卡复位 | —(主控证据不受影响) | — | 线卡本地小保留区(若硬件有)存执行插件状态 |

- **保留 DDR 的 BSP 需求清单(交付依赖,列入 §14,按平台滚动交付):** bootloader 对预留段跳过内存初始化与检测;DDR 重训练策略确认预留段内容保持;ECC 平台对预留段采用"保留内容 + 重启后校验清洗"或非 ECC 映射;内核保留内存映射;协议启动前先行挂载并做完整性扫描(逐条 CRC,丢弃半写条目)。
- **硬看门狗复位路径的两级缓解:**
  - (a) 平台支持复位前不可屏蔽中断/预警的,注册快照最近事件到片内 SRAM;
  - (b) **不保 DDR 的平台上,SRAM 微缓冲采用逐条增量写**:每条接入事件的 {时戳, 邻居, 协议, 关键头字段紧凑编码 ≈16–24B} 直写 SRAM 小环,并在 SRAM 内划出小钉住区(循环模式下候选类样本免逐出)。这保证"崩溃前最后一条接入"必然在 SRAM——对 C3 这类"真凶与崩溃间隔微秒级"的缺陷,周期性镜像必然错位,只有逐条写才守得住。SRAM 写带宽平台相关:带宽不足的平台可按协议/上送类别过滤只记高危面(如 LDM 上送类),仍不足则如实降级回周期镜像(约 100ms 一次镜像各环写指针 + 最近指纹),**并在 §13.4 声明:该平台组合下,硬看门狗复位循环中的 C3 类真凶大概率无法取到**。逐条/过滤/周期三档为平台配置项(§12)。
- **明确的载体决策:** NVRAM/Flash 不作热路径日志载体(带宽与寿命不支持),仅承载免疫库、事件档案、标定数据、稳态基线快照(慢路径)。

### 5.2 双日志与统一时间轴

黑匣子记两条日志,经 §2.5.2 的时序基准归并成同一条时间轴——这是因果核对、跨事件对照对齐、发作路径归因的底座:

- **接入日志:** 追加写、纯环形、严格按时间老化。每条含 `环内序号 / 时戳 / 入接口 / 邻居 / 协议类型 / 报文指纹 / 关键字段紧凑拷贝 / 处置结果(1B)`。**指纹的构成显式定义:归一化类键哈希 + 变化字段哈希两段**——类键哈希使指纹级条目也能按归一化类参与跨事件对照统计(收敛期被降级为指纹的条目不失去统计价值);新奇或高危报文存全量(受 §5.4 配额)。**处置结果由协议经 record_verdict 回填**(条目在报文处理结束时一次性发布,结果内联)——这是"该类报文本就被策略丢弃"这一处置等价性证明(§6.6)的证据来源:证明靠关联既有处置记录,不靠理解丢弃策略语义,与黑箱立场自洽。
- **状态变更日志:** LSDB/RIB 增删、状态机迁移,每条**内联其成因的溯源号**(§2.5.2 因果证据 a);支持批量聚合条目("邻居 X 批量安装/撤销 N 条,首末序号"),收敛期不逐条。

### 5.3 溯源摘要仓

定长槽位 `{溯源号, 协议, 归一化关键字段拷贝(64–128B), 首见, 最近引用, 免逐出位}`;打标签时写入/刷新(仅创建过驻留对象的报文才占槽,天然筛掉纯过路流量);近似最久未用淘汰;硬上限缺省 32MB;失引语义见 §2.5.1。**免逐出位:** 归因支点对象、绊线关联对象、NSR 跨板同步来的摘要、循环模式下的候选类样本置免逐出,免逐出配额 ≤ 仓容量 25%,超限按证据档位再逐——保证"最需要溯源的那些摘要"不被日常路由变动冲掉。与时间轴同处保留区,逐槽 CRC,复位存活等级同 §5.1。规模示例:OSPF/IS-IS 逐 LSA/LSP 一槽,10 万 LSA × 128B = 12.8MB;BGP 打在引用计数共享的属性集上(属性集数量远小于路由数,粒度对归因结论无实质影响),均在 32MB 内。

### 5.4 自适应保真度与抗冲刷

- **冷启动 / 批量收敛抑制门:** 会话建立、GR、整表收敛等**有相称原因的批量期**,新奇判定挂起,全部只存指纹——堵死"新设备上线基线为空、全表判新奇、瞬间冲垮窗口"。
- **全量捕获配额:** 每源邻居 ≤ n 条/s 与 ≤ m KB/min,全局预算 ≤ 环容量 25%;超限降级为指纹。"新奇洪泛"的写放大被配额封顶。
- **疑犯钉住配额区(固定 8MB,独立于普通环):** 命中绊线关联、高危、或已成为归因候选的报文全量样本存此区,普通新奇流量**挤不掉**;逐出需 PAD 确认。持续洪泛下"每次复发真凶也被同速冲走"的失效模式由此消除。
- **冲刷即绊线:** 新奇率异常升高本身注册为一条绊线告警("疑似证据冲刷攻击")。
- **循环模式联动:** 崩溃循环检测(§6.1)自动抬高保真度并优先写钉住区——首次异常真凶只留粗指纹,复发即抓全量,与"首受害者中招、之后免疫"一致。

### 5.5 容量核算:收敛期最坏情形

"窗口 × 速率封顶"只在稳态成立,收敛期速率正比于路由规模,按最坏情形核算窗口覆盖:

- 100 万路由单邻居全表收敛,抑制门生效(全部指纹级 48B/条),更新粒度约 25–50 万条 → 接入日志 12–24MB(热路径报文环容量下限 16MB 由此而来);状态日志批量聚合后 KB 级。
- 缺省 128MB 时间轴 ⇒ 可容 **≥5 个完整收敛窗口**;双栈 × 多邻居按线性折算,逐机型出厂给出"最坏收敛速率 → 窗口时长"表(§12 治理)。
- 崩溃循环场景:每个崩溃周期窗口 ≥1 个完整收敛,且候选真凶进钉住区不占普通窗口——对照统计所需的多窗口保留成立。
- 介质写带宽:保留 DDR 为内存写,无 Flash 寿命问题。

### 5.6 数据结构与内存核算

见 §2.5.1 单一权威核算表。**结论:全程没有第二份路由数据;唯一与对象数相关的结构(溯源摘要仓、溯源标签)分别有硬上限与逐对象 8B 的极小本体。**

## 6. 归因定位

归因是整个系统里唯一"难"的部分。它不是一个算法,而是一个**从"黑匣子里记的所有东西"漏斗式收敛到"1–3 个候选 + 为什么是它们 + 发作在哪一侧"的过程**,全程不重放,并在出闸门前经排除验证把"统计罕见"与"因果致病"分开。

```
 异常触发 ─► ①入口分类(支点+窗口+循环检测)─► ②并行缩小 ─► ③融合+证据档位 ─► ④判别字段
                                             ├ A 溯源链追溯                       │
                                             ├ B 驻留态异常扫描          ⑤排除验证(§6.4)
                                             ├ B′ 回放态扫描(让位窗口)           │
                                             ├ C 跨事件对照统计          ⑥出口闸门(分层,§6.6)
                                             └ D 速率/重复模式                    ├► 出免疫条目
                                                                                  └► 内部缺陷:自愈+上报
```

### 6.1 入口分类

触发器一响,第一件事是把异常翻译成一次具体查询:一个**支点** + 一个**搜索窗口**;这依然是对发作形态分类,不是对根因分类。

| 触发形态 | 支点 | 走向 |
|---|---|---|
| 卡死 | 状态机卡住的那个对象 | A 溯源链(跨板经摘要仓,§2.6) |
| 批量删 + 因果核对 | 被删前缀集(ΔLSDB≈0 且无相称撤销 ⇒ 计算错) | B 驻留态扫描 + ⑤排除验证 |
| 崩溃(常规:崩溃前窗口内有正常接入流) | 时间游标 | C 对照统计 |
| 崩溃于回放/首算期,且窗口内新增接入 ≈ 0(回放循环) | 回放出的驻留态 | B′ 回放态扫描 |
| 报文风暴 | 风暴对象类 + 速率模式 | D |

**崩溃循环检测与让位窗口:** T(缺省 30 分钟)内 ≥2 次复位 → 进入循环模式:抬保真度、启用钉住区,并在下一周期**回放完成后、首次全量计算开始前**,由框架向协议请求**暂缓首次计算 ≤ T_scan(缺省 5 秒,§12)**,先执行 B′ 回放态扫描——此时状态回放的写入已结束、首次计算尚未开始,**没有计算写锁争用,扫描窗口是设计保证的而非碰运气**。协议经能力注册声明是否支持暂缓;不支持、扫描超时、或回放期本身反复崩溃致扫描无法完成 → 保护性冷启动兜底(§8.6)。非循环状态不插入让位窗口。

### 6.2 并行缩小器

由入口分类决定跑哪几路,各吐候选 + 理由。全部是通用机制,不含逐缺陷知识。

**(A) 溯源链追溯。** 顺着对象上的溯源标签:卡死的下一跳 → 路由 → 属性集 → 那条更新报文 → 邻居,O(1) 指针追溯,精度最高。跨板经 NSR 随对象同步的摘要在本板解析(§2.6,免逐出位保护);摘要被淘汰时按 §2.5.1 失引语义降级到 B/C,不失败。C2 的兜底归因走这个(主路径在入口能力校验)。

**(B) 驻留态异常扫描。** 对付时间解耦的最重要一招:**真凶报文往往几小时前就来了,发作时早已滚出时间轴窗口——但它变成的那个对象还躺在 LSDB 里。不翻历史报文流,扫当前驻留状态,就地扫毒。** 打分 = 长窗基线罕见度 + 规范先验 + 首见年龄(§2.5.5,训练卫生保证毒化对象不会把自己刷成基线)。O(N) 一次、只在触发后跑。C1 走这个。
**执行协议:** PAD 下发**声明式打分规格**——字段列表 + 各字段分布摘要快照 + 权重,纯查表与算术,无任意代码、无循环、无副作用(不引入代码注入面);协议 worker **分片遍历**:每片 ≤4096 对象、片间释放锁并让步、单片 ≤2ms、总预算超时即中止;**锁纪律:worker 走普通读者加锁路径**——若某线程恰持 LSDB 写锁导致遍历超时,则如实上报"驻留扫描不可用",PAD 降级到时间轴路径,不死等不死锁;回传 top-K 一律标注**"由受怀疑进程自报"**,PAD 对可得的摘要仓/时间轴记录交叉核对后方进入融合。

**(B′) 回放态扫描。** 回放循环下,毒化对象恰在回放出的 LSDB/RIB 里。B′ 在让位窗口内(§6.1)对驻留态执行与 B 完全相同的打分;命中对象的溯源标签反查摘要仓(若在)补齐"何时从哪来"的证据。**匹配条件来自对象自身归一化字段——摘要失引不阻断出条目。** 让位窗口使 B′ 在循环模式下成为有保证的主路径;窗口不可得时冷启动兜底。走查见 §10.2。

**(C) 跨事件对照统计。** 利用崩溃循环本身:重复样本让不变量显形。设计要点:

- **等价类定义(决定成败):** 归一化类键 = {协议, PDU/对象类型, 按 §7.2 同一套泛化/掩码规则处理后的判别字段值}——与免疫条目泛化**共用同一机制**,天然对齐"同一毒化对象每次字节都不同"的问题。指纹级条目携带类键哈希(§5.2),收敛期数据同样可参与统计。
- **判据 = 对照提升度:** 对每个类 c 计算 lift(c) = 出现率(各崩溃前窗口) ÷ 出现率(等长对照窗口)。周期性背景报文(Hello/保活/周期刷新)两侧出现率 ≈1、提升度 ≈1,被自然压掉。**对照窗口来源分级:** 优先取同一时间轴的非崩溃期;**密集循环下正常窗口稀缺时,取循环开始前持久化的最近稳态基线快照**(§2.5.5)——对照统计在其最该发力的高频循环场景不再因"没有正常窗口"而失效。快照也不可得(如首轮循环即冷机)→ 如实声明 C 路退化、由 B/B′ 与冷启动承接,并计入度量口径(§15"C 路退化事件")。
- **资格门与排序分离:** 提升度只作**资格门**(lift ≥ θ_lift 且 ≥2 次崩溃可算、≥3 次置信提升,候选上限 K=10);入围候选交 §6.3 统一加权融合排序——避免"提升度 × 异常分"乘积把"低异常分 × 高提升度"的真凶(正是 C3 型合法报文命中缺陷)挤出候选,而把"高异常分 × 提升度≈1"的新奇合法报文顶上去。资格与排序各司其职:C3 型靠提升度入围,靠处置等价性证据(§6.6)成臂,不需要赢排序头名。
- **退化路径:** 交不出高提升度候选、对照不足、或候选爆炸 → 输出"低置信转人工 + 保真度已抬高等待复发",不硬出条目。窗口小于"到达 → 发作"时延的延迟发作类,C 结构性够不着 → 由 B/B′ 承接(毒在驻留态)或如实低置信。

C3 走这条(真凶 = 命中丢弃路径的 IPsec 报文类,提升度极高;每次变化的无辜受害报文与周期背景都被对照压掉)。

**(D) 速率 / 重复模式。** 高频对象统计锁定的风暴对象 + 速率轨迹即候选,走行为处置(§8.6)。

### 6.3 打分融合与证据档位

多路候选融成排序。这里是透明的加权打分,不是黑盒模型——证据链必须能被运维看懂,可解释性优先于准确率。信号:

- **异常分:** 字段取值相对**长窗**基线的罕见度 + 规范先验(值域级依据)。
- **能力依据:** 是否落在支持矩阵之外(确定性依据,§3.2)。
- **溯源强度:** 是否被标签直连到发作对象。
- **复现度:** 对照提升度。
- **因果邻近:** 候选是否落在受影响前缀的计算依赖锥内(硬检查项,§6.4)。
- **新奇度:** 首见年龄。**新奇度只参与排序,不构成任何破坏性处置的依据**(§6.6)。
- **触发条件吻合:** 进入时间与计算计数轨迹等特定条件是否对得上。

**决策不绑数值置信度,改绑离散证据档位:**

| 档位 | 定义 | 允许的自动化 |
|---|---|---|
| E3 | 溯源直连 + 排除验证判别成功;或能力失配(确定性依据,处置限定义化降级) | 自动进观察态;人工确认后可执行态(含破坏性处置;定义化降级项经确认可直接执行态) |
| E2 | 提升度显著(≥θ_lift)且 ≥3 次独立发作命中该条件;或 E1 + 排除验证通过;或处置等价性成立(§6.6);或值域级明确异常 + 算错裁决确证本机算错(排除验证无判别力时的 C1 路径,§6.4) | 处置非破坏时自动进观察态;执行态需人工 |
| E1 | 单次发作 + 统计显著异常 | 仅生成候选档案,人工决定是否进观察态 |
| E0 | 仅统计罕见 / 低置信 | 只上报,不出条目 |

数值置信度保留为辅助显示。**标定闭环**(网管侧汇集全网〈候选, 人工裁决〉对拟合可靠性曲线,标定器版本化随免疫库分发;冷启动样本由 §14 缺陷注入测试床回放生成)达到样本量门槛前,数值不得绑定任何自动决策。"没有一个候选分高时输出低置信转人工"的原则始终成立。

### 6.4 排除验证:把"罕见"与"致病"分开

**"罕见"不等于"致病"。** LSDB 中罕见且完全合法的对象大量存在(少见可选 TLV、全网仅一台始发的链路类型),十万条 LSA 的 0.1% 尾部就是上百个高分对象;而因果核对为真、裁决确认"算错"为真,都不证明"被点亮的那条 LSA 就是原因"。出闸门前增加一步廉价的因果检验:把头号候选从输入中排除,重算一次,看症状是否消失。

**两类执行体,职责分开,不可混用:**

- **生产实例的受保护试算(首选执行体)。** 熔断冻结期间、GR/NSR 保护下,生产实例排除头号候选后重算一次(限一次)。生产实例**自带缺陷与全部内部状态**,判别力最高:排除后症状消失(误删路由恢复/结果转正)→ 因果成立;排除与否结果相同 → 该候选降级、顺延下一个。这与 §8.6 熔断恢复流程中"拒绝应用删除、重启计算"天然合一:重启计算时先排除头号候选即可,不增加独立风险(计算重启本就是恢复动作,且处于冻结保护中)。
- **生产同码影子(次选执行体)。** 与生产完全同版本的代码在隔离实例(备板空闲核/容器)上运行,用于生产实例不宜试算的场景(如倒换敏感期)。同码保证缺陷忠实。
- **独立参考计算器不做排除验证。** 它是与生产无关的极简正确实现(§8.6),职责是裁决"本机是否算错"。一个正确的实现不含生产缺陷,排除候选与否它都算出同样(正确)的结果——对"输入触发实现缺陷"型问题**没有判别力**。两个组件的正确性要求相反(一个要忠实于缺陷,一个要不含缺陷),必须是两个组件。

**适用边界(如实声明):** 排除验证对"输入改变正确计算结果"型(语义/拓扑致因)判别力强;对"由累计内部状态触发"的实现缺陷型(如 C1 的计算次数阈值类),即使同码试算也未必复现——计数器状态在重启计算后可能已不同。**试算不复现症状(无论是否排除候选,结果都正常)时,输出"无判别力",既不确认也不否定候选**,证据链退回:值域级异常 + 独立参考计算器的算错确证 = E2(§6.3),破坏性处置按分层闸门需人工或第二次发作佐证。C1 的证据档位因此是条件式的:试算判别成功 → E3;无判别力 → E2 + 人工,走查 §10.1 按此如实描述。

**因果邻近硬检查:** 候选必须落在受影响前缀的计算依赖锥内;不在锥内的候选不得仅凭罕见晋级。

### 6.5 判别字段提取

对头号候选,不止说"是这个报文",还要说"是它的哪几个字段异常"。逐字段异常分直接给出:值本身异常的字段(ls_id=0)就是判别字段,进匹配条件;天然变化的字段(LS Age、序列号、校验和、源 IP/MAC)按构造掩码掉——这就是不做重放的前提下、替代"逐字段消融重跑"的机制。条件最终宽度由处置安全性联动决定(§7.2)。边界照旧诚实:复杂跨字段语义条件自动模型抓不全,交人工收窄。

### 6.6 出口闸门(分层判据)

免疫条目出闸的判据按处置的破坏性分层,并设一条机械前置,防止"统计上新奇"被当作拦截依据:

- **机械前置(适用于一切破坏性处置):** 判别字段必须存在**值域级异常依据**——规范违例、能力失配、或长窗基线下的值域显著罕见——三者居其一。**仅有"首见/新奇"的输入,无论发作在哪一侧,一律封顶观察态。**"合法但新奇的输入诱发内部缺陷"由此被机械地挡在破坏性处置之外,归自愈轨道。
- **单臂即可出条目(观察态起步)的三种依据:**
  - 非破坏处置(观察;隔离且守护证明本机转发路径不变);
  - **处置等价性:** 由处置结果记录证明该归一化类历史处置 100% 为策略丢弃(证明条件与策略版本绑定见 §7.1/§7.5)——"提前丢弃本就要丢的"结果一致、误杀为零;
  - **能力失配:** 支持矩阵给出确定性依据,授权对应的**定义化降级**处置(按撤销处理/忽略不选路)——处置正当性来自"本机无法正确处理"这一可验证事实,不需要"该输入有害"的统计推断。
- **破坏性处置**(按撤销处理〔非能力依据的〕/ 丢弃 / 降级 / 影响算路的隔离):在值域级异常之外必须附加**因果证据**——排除验证判别成功,或该条件在 ≥2 次独立发作中均命中且隔离后不复发(证据档位 ≥E2 且含因果项)。
- 无因果证据 → 最多出观察态条目 + 人工。
- **档案语言规范:** 归因结论字段固定表述为**"与发作强关联的输入"**,不得写"原因";**"替代假设"为必填字段**(至少含"内部缺陷被合法输入诱发"),供审核人显式排除后才可晋级破坏性处置。

案例对照:

| 案例 | 路径 | 档位 | 闸门结论 |
|---|---|---|---|
| C1 | B 驻留扫描;排除验证条件式 | E3(试算判别成功)/ E2(无判别力 + 算错确证) | 值域异常成立 → 出值特征;处置=隔离(计算侧)+ 一致性守护;E2 时执行态需人工或二次发作 |
| C2 | 入口能力校验(主);A 溯源(矩阵缺项时的兜底归因) | E3(能力失配,确定性) | 定义化降级(按撤销处理)单臂成立;矩阵缺项时事后产出能力补充项,人工确认入矩阵 |
| C2 新奇变体(该机首见某类合法路由,统计"新奇"但在支持矩阵内) | — | E1 封顶 | 机械前置:仅新奇不构成破坏性依据 → 最多观察态 + 自愈兜底 |
| 合法新奇 LSA 触发**计算侧**内部缺陷(支持范围内) | B 扫描点亮 + 排除验证可能通过 | 受机械前置约束 | 无值域级异常 → 封顶观察态,转自愈——"合法输入 + 内部缺陷归自愈"由机械规则保证,不依赖审核人自觉 |
| C3 | C 对照统计提升度极高 + 处置结果记录证明"本就策略丢弃" | E2 | 处置等价性单臂成立 → 出条目(提前 CoPP 丢,零误杀);条件与丢弃策略版本绑定 |
| C4 | D 速率模式 | — | 行为处置(§8.6 计算视图暂缓),不出报文值特征 |

### 6.7 发作路径归因(为处置选择供给)

给崩溃/异常定位**发作路径类别**并写入证据档案,依据是 §2.5.2 的两条显式因果证据,不依赖跨环时间比较:

- **崩溃线程定位:** 崩溃线程独占环的末尾条目是接入事件,且该线程环内其后无状态变更条目 → **接收/解析/安装侧**(报文处理中崩溃);
- 崩溃发生在计算 worker 线程(其环末尾是"计算开始"类条目,起止同环全序可靠),接入侧安静 → **计算侧**;
- 卡死 → **状态机/内部侧**。
- 边界情形(崩溃线程不明、末尾条目歧义)→ 发作路径标"不确定",处置选择保守(取交集安全的处置或转人工),**不猜**。

该分类直接约束处置选择(§8.3):同一条目命中,计算侧缺陷配"隔离出计算",接收侧缺陷配"骨架入库";发作路径与处置不匹配的条目被入库校验器拒绝(§7.3)。

## 7. 免疫条目生成与持久化

### 7.1 免疫条目结构

```json
{
  "id": "sig-2026-0718-0001",
  "kind": "learned_value | capability_supplement | verdict_equivalence",
  "protocol": "ospf", "pdu_type": "LSU/Network-LSA",
  "predicate": { "ls_id": "0.0.0.0", "mask": ["ls_age","seq","checksum","src"] },
  "fault_path": "compute | ingest | fsm | unknown",
  "disposition": "observe | sanitize | defined-degrade | treat-as-withdraw | isolate-from-computation | skeleton-ingest | rate-limit | drop",
  "scope": { "vrf": "*", "instance": "*", "interface": "*", "neighbor": "*", "area": "*", "afi_safi": "-", "global": false },
  "policy_ref": null,
  "evidence_tier": "E0-E3", "confidence_display": 0.93,
  "dossier_ref": "flash:/pad/dossier/0001.json",
  "defect_id": "DTS2026xxxxx", "fixed_in_version": null,
  "state": "observe | enforcing | suspended | retired",
  "hit_count": 0, "last_seen": null,
  "ttl": "180d", "review_after": "90d",
  "origin": "local | nms", "signature": "<HMAC/签名>"
}
```

- **kind 三类:** 学习值特征(C1 型,统计+因果证据支撑);**能力校验补充项**(C2 型,事后归因发现支持矩阵遗漏,人工确认后并入矩阵,处置限定义化降级);处置等价类项(C3 型,处置结果记录支撑,**policy_ref 必填**——记录其依据的丢弃策略标识/版本指纹)。
- **ttl 必选**(缺省 180 天,命中自动续期;infinite 须显式声明并强制 review_after 复审)。**defect_id/fixed_in_version:** 关联缺陷单;设备升级到修复版本后条目自动转 suspended 待人工复核退役——避免"缺陷已修复 + 条目永久隔离"的持续不一致,并给补丁留验证窗口。
- **证据档案(dossier)必含:** 触发事件、时间线摘录、各路归因输出、排除验证结果(含"无判别力"结论)、发作路径依据、**影响面预估**(该条目会命中的合法流量画像 + 处置的路由影响预估:隔离类含计算依赖锥,守护联动类含**预估引流面**——§13.1 回滚判据依赖它)、**替代假设(必填)**、建议处置与生效范围。

### 7.2 泛化与匹配条件宽度联动

- **掩码天然变化字段**(LS Age/序列号/校验和/瞬时字段);判别字段带值或值域;单条目谓词数上限(缺省 8)。
- **宽度随处置安全性联动**,方向有意相反:
  - **非破坏处置(观察/告警/取证):允许宽域与语义式条件**——值域条件("ls_id ∈ 保留域")、关系条件("Network-LSA 与任何 Router-LSA 无双向对应"),取自规范先验表与不变式素材,覆盖缺陷触发域而非单点。逃逸空间显著压缩。
  - **破坏性处置:维持最小匹配条件**(误杀优先)。由此保留的逃逸空间是清醒的取舍,列入固有局限(§13.4)。
  - **变体扩展流程:** 观察态宽条目命中 + 与既有执行态条目同 defect_id 关联 → 自动提出"条件并集扩展"复评,经人工确认后收窄执行条件的逃逸面。

### 7.3 入库校验器(机械执行,不依赖自觉)

每个条目入库前过静态校验器,拒绝项:

1. **红线校验(§8.2 双轴):** 效果轴为根本判据——处置会丢弃/改写/延迟/抑制任何会话建立、保活、确认、数据库同步类报文本身,或改变同步/会话状态机的标准迁移 → 拒绝。对象轴为缺省实现:按 PDU 类型整类匹配同步类报文 → 拒绝;允许匹配同步报文**承载的载荷对象**(LSU 中的 LSA、UPDATE 中的属性/NLRI);同步报文中承载的**头摘要**(DD 中的 LSA 头、CSNP 中的 LSP 条目)允许**只读/观察/取证类**匹配(如标记该对象"到货后按既有条目处置"),禁止任何据此跳过请求/确认/重传等同步动作的处置。
2. **发作路径—处置匹配:** fault_path=ingest 的条目配 isolate-from-computation → 拒绝(隔离对接收侧缺陷无效);fault_path=unknown → 仅允许两侧都安全的处置或观察态。
3. **宽度—处置联动校验**(§7.2);**scope 完整性**(实例化协议必须携带 vrf/instance 上下文);**影响面预估存在性**;**能力补充项的处置限定义化降级**;**等价类项的 policy_ref 存在性**。

### 7.4 持久化与加载

- 免疫库存 Flash,双份 + 版本 + CRC + 完整性校验;**本地条目与网管条目同等强制完整性**:网管下发的带中心签名;本地学习的以设备唯一密钥(有 TPM 用 TPM)做 HMAC。启动时逐条验证,**被篡改/损坏的份拒载,用另一份**。
- **双份均校验失败:** 缺省**空库放行**——协议恢复不被阻塞(防护系统不成为收敛单点),持续最高告警,并在管理通道可达后**自动向网管请求重新下发免疫库**(网管保有审核后的全网副本,恢复时间为管理通道级的分钟量级);可配置**严格档(fail-secure)**:双份失败时拒绝该协议自动恢复/降级本机路由角色,供高安全场景选用。**如实声明:空库窗口内,重收型死循环没有入口拦截可用**(保护性冷启动只兜回放型),该窗口的攻击价值与恢复依赖已列入 §13.2/§13.4。
- 加载时序与"先于回放生效"由协议侧框架自主完成(§2.5.3);复位后立即执行的仅限执行态条目。

### 7.5 生命周期治理

- **状态机:** observe → enforcing(人工审批)→ suspended(误报证据 / 修复版本联动 / **所依赖丢弃策略变更**〔等价类项,policy_ref 失配即自动挂起复核〕/ 逃生开关)→ retired(ttl 到期 / 人工 / 复审不通过)。
- **数量上限:** 本地自学 ≤512,含网管下发总量 ≤4096;超限按 (evidence_tier, hit_count, last_seen) 逐出,逐出即告警。NP/TCAM 下压子集独立预算,超限退回软件判定。
- **重叠仲裁:** 入库时做条件蕴含分析;同对象域多条目命中按 (evidence_tier, priority) 取最高,处置冲突时**更破坏的处置需更高审批等级才可胜出**。
- **等价类项的成立与维持条件:** 处置结果观测窗 ≥7 天且覆盖策略/会话状态迁移(如 IPsec 类须含 SA 建立后的窗口——状态相关的丢弃在会话建立后会转为接受,循环期采样"100% 丢弃"不能外推),或人工确认该丢弃策略无状态相关;policy_ref 指向的策略发生任何变更 → 条目自动 suspended 待复核。
- 全部增删改、状态迁移、命中统计进审计日志。

### 7.6 加载即生效的驻留清理(限计算侧)

免疫条目加载后,除入口匹配外,对**已在库**的对象做一次驻留扫描:fault_path=compute 的条目命中驻留对象 → 隔离出计算(过 §8.4 守护)——打断回放型死循环的关键动作之一;fault_path=ingest 的条目不做驻留隔离(缺陷在解析/安装路径,对象已入库,隔离无意义),标记观察。

### 7.7 LDM(链路层报文处理模块)专项

- **受限 IR:** {L2/L3/L4 头字段, 上送原因码, 目的地址类别}——有限、定长、可静态判定;字段分布基线仅对该受限域训练。
- **条目子集:** 纯头部匹配条件;处置以提前丢弃/CoPP 限速为主(LDM 场景丢弃语义安全——上送面报文无路由数据库同步语义,C3 即此类)。
- **硬件编译约束:** 条件 → NP/ACL 表项仅支持定长偏移字段;不支持流重组、变长 TLV 深度解析;不可编译的条目回落主控软件判定并标注。

## 8. 分级处置与自愈

### 8.1 处置阶梯

破坏性从低到高;每级适用约束与撤销路径明确:

| 级 | 处置 | 破坏性 | 适用约束 |
|---|---|---|---|
| 0 | 观察/告警/取证 | 无 | 任何条目的缺省起点 |
| 1 | 净化(属性丢弃/字段裁剪) | 低 | 违例字段可剥离且协议定义了剥离语义 |
| 1′ | **定义化降级**(能力失配:按撤销处理/忽略不选路/属性丢弃) | 低* | 仅能力依据(§3.2/§6.6);*本机本就无法正确使用该输入,"不用"比"误用"安全 |
| 2 | treat-as-withdraw(值异常依据) | 中 | BGP 类;需因果证据(§6.6) |
| 3 | 隔离(计算侧):入库+泛洪+不参与本机计算 | **中** | fault_path=compute;**必须过 §8.4 一致性守护** |
| 3′ | 骨架入库(接收侧,§8.3) | 中 | fault_path=ingest;**必须过 §8.4 一致性守护** |
| 4 | 限速/延迟处理 | 中 | 风暴类;上限见 §8.6 节奏控制 |
| 5 | 丢弃 | 高 | 仅 LDM 上送类 / 处置等价性成立的类;路由数据库对象原则禁用 |
| 6 | 关 AFI/SAFI、会话级动作、设备级引流 | 很高 | 仅人工,或 §8.4 守护联动(自动但显式告警) |

隔离(计算侧)的破坏性定为"中"而非"低",因为它使本机计算输入偏离域内共同视图——这正是 §8.4 存在的理由。

### 8.2 硬红线(不可越)

- **效果红线(根本判据):** 任何免疫条目的处置,不得丢弃/改写/延迟/抑制**会话建立、保活、确认与数据库同步类报文本身**(Hello/IIH、DD、LSR、LSAck、CSNP/PSNP、OPEN、KEEPALIVE、NOTIFICATION、ROUTE-REFRESH,及经 LDM 的 BFD/LACP),不得改变这些交换所属状态机的标准迁移。对这些报文允许的只有:观察、取证、以及对其**承载的载荷对象**(LSU 中的 LSA、UPDATE 中的属性)按本系统机制处置。
- 学习条目只允许作用于承载路由/拓扑信息的**载荷对象**与 LDM 上送头(对象红线,§7.3 机械执行)。
- 隔离的对象**继续参与数据库同步与确认**(隔离 ≠ 从数据库消失)。
- 自动处置必须可回滚;越阶梯生效需人工。

### 8.3 处置匹配发作路径

同一条目命中,正确处置取决于缺陷发作在哪一侧(§6.7 供给判定):

- **计算侧(C1 型):** 隔离出计算(数据库/泛洪/确认照常)→ 打断"回放 → 计算 → 崩溃"循环。过守护(§8.4)。
- **接收/安装侧:** 隔离无效(下次崩溃仍在解析里),丢弃有害(邻居数据库认为本机该有此对象,重传不止、对账失配)。处置 = **骨架入库**:仅解析对象头(约 20B,LSA 头/LSP 头,身份/序列号/寿命/校验和字段)用于数据库身份、DD/CSNP 对账与确认;校验和按原始字节算术校验(不需体解析);体 TLV **不解析、不安装**;泛洪/重传直接使用保留的原始字节。脆弱的体解析代码不被执行,同步语义完整。**骨架入库使该对象内容不进本机计算图,属计算输入偏离,必须过 §8.4 守护。** 前提如实:头解析路径本身健壮(20B 定长,可穷尽测试)。
- **头解析 / 泛洪路径自身的缺陷:** 骨架入库也救不了(缺陷在头/泛洪代码)。如实承认**没有非破坏解**,能真正止住循环的选项只有:(a) 拆除相关邻接/关闭接口(显式破坏同步与拓扑,人工决策;可先经 §8.4 联动引流再拆,减小业务冲击);(b) 入口按对象标识丢弃该特定 LSP/LSA(**显式承认破坏可靠泛洪**:邻居重传持续、CSNP 对账失配,仅作人工授权的短期止血,并配 CoPP 吸收重传冲击);(c) 离线补丁。**overload/max-metric 不在此类处置之列——它们只改变计算与过境角色,完全不停止泛洪与报文接收**,置位后本机仍收该报文、仍走脆弱代码。
- **FSM/内部侧(C2 型):** 条目/能力项的价值在入口预防(下次直接定义化降级,不进卡死路径);对当前卡死走看门狗恢复(§8.6)。

### 8.4 路由计算输入一致性守护(横切规则)

**不变式:链路状态协议的正确性建立在"域内所有路由器基于相同 LSDB 计算"之上;任何使本机计算输入偏离标准邻居的处置,都可能制造持久转发环或黑洞——它不同于收敛期的瞬态微环,会稳定存在直到处置撤销。** 本节是**所有**此类处置的统一前置门,无一例外:

**受控处置清单(凡改变本机计算输入的,全部在列):**

| 处置 | 计算输入偏离形态 | 允许的守护形态 |
|---|---|---|
| 隔离(计算侧) | 对象被排除出计算 | 免联动证明(a)(b) 之一,或强制联动 |
| 骨架入库 | 对象内容不可入计算 | **免联动证明不可运行**(证明 (a) 需体内链路信息、证明 (b) 需对象内容,骨架均无)→ 仅"依赖锥判定 + 联动"一条路 |
| purge/风暴对象的计算视图暂缓(§8.6) | 单对象冻在旧版本,其余更新照常 → 部分不一致输入 | 依赖锥内:T_hold 压至 ≤10s(与计算去抖同量级)或强制联动;锥外:允许 60–300s |
| 抑制/暂停某对象泛洪(仅人工短期止血,§8.3) | 邻居与本机数据库失配 | 人工授权 + 强制联动(引流后执行)+ 显式代价记录 |

**守护判定与形态:**

- **免联动证明(满足其一可单独处置,无需引流):**
  (a) 该对象在标准计算中本就不可达/不被采用(如 Network-LSA 双向连通性校验失败——正确的计算本就不会把它纳入最短路径树),本机排除与邻居纳入结果相同;
  (b) 生产同码影子证明:排除该对象后,本机作为**过境节点**的全部转发路径不变(仅影响以该对象为目的的末梢前缀)。
- **依赖锥判定(骨架入库等无法运行证明时的准入依据):** 按**始发路由器粒度**——用当前 LSDB 判断该对象的始发者是否位于本机任何过境最短路径上(头部含始发者标识,无需体解析)。不在任何过境路径上(末梢始发者)→ 允许单独处置,影响限于该末梢前缀可达性;在过境路径上 → 必须联动。
- **强制联动:** 处置生效期间,本机同步置 IS-IS overload / OSPF max-metric(RFC 6987)/ BGP GRACEFUL_SHUTDOWN——主动退出过境角色,让邻居绕开本机,把"隐蔽的持久环路"转换为"可观测、可回滚的过境角色退出"。联动状态随处置生命周期同步撤销。
- **语义如实(不夸大守护能力):** IS-IS overload 使邻居不经本机过境(直连仍达);OSPF max-metric 仅把过境链路代价置最大,**当本机是唯一路径(割点)时仍会被用作过境**——守护的免环保证以**"存在替代路径"为前提**。割点/无替代路径场景:守护无法生效,而此时依赖锥必然巨大 → 触发影响面挂起线(§12)转人工——把割点场景显式导向人工,而不是假装守护有效。
- **影响面预估必含依赖锥:** 受影响的下游前缀数与过境流量画像进入证据档案;守护联动类同时写入**预估引流面**(§13.1 回滚判据的基准)。超阈挂起转人工。
- **每台设备同时处于守护联动态的对象数有上限**(缺省 4,§12),超限一律转人工——防止多条目联动叠加把设备打成长期非过境。

### 8.5 网络扩散决策(显式取舍)

**隔离类处置只保护本机,不阻止毒化对象经本机扩散**——继续泛洪是数据库一致性的代价,如实声明而非掩盖(§13.4)。补偿机制:

- **补偿一(压缩免疫窗口):** 特征命中即最高级告警;网管加速审批通道——签名、灰度、可回滚一个不减,只压时限(详见 §9)。
- **补偿二(人工升级,显式代价):** 对已确认的崩溃类毒化对象(证据档位 ≥E2、邻居中同型号占比高),审核人可选择设备级升级:先经 §8.4 联动引流(退出过境),再**拆除通往未防护下游的邻接/关闭接口**以阻断经本机的扩散——这是显式承认"切断同步换取止损"的人工运营决策,代价(该方向拓扑收缩)记录在案;不存在"置 overload 就能停止泛洪"的选项(§8.3)。
- 扩散事件(免疫窗口内经哨兵泛洪导致下游设备发作)计入度量(§15)。

### 8.6 自愈动作集

- **删除熔断与解冻:** 触发即冻结(§4.2(3))。解冻三出口:① 算错裁决确认"本机算错" → 拒绝应用该批删除、排除头号候选后重启计算(与排除验证的受保护试算合一,§6.4)→ 恢复即解冻;② 裁决确认"输入如此" → 应用删除,解冻(是合法收敛);③ **裁决不可得/超时 T_max(缺省 10 分钟):保守退出**——维持"不应用可疑批量删除"的冻结侧,升级告警,等待经认证的人工;**绝不自动应用未经验证的批量删除**。可能的黑洞窗口由 T_max 封顶并显式告警。裁决能力交付前,熔断=冻结+告警+人工,如实计入 §14。
- **算错裁决(独立参考计算器):** 与生产无关的极简正确实现(基础 SPF + 双向连通性校验),仅回答"本机计算结果是否偏离正确结果",**不做排除验证**(§6.4)。
  - **同快照协议:** 熔断时经服务请求导出带版本戳 v 的 LSDB 快照;主实现待应用删除标注其计算轮次的输入版本 v_main;**v_main ≠ v 或导出期间数据库变更超 ε(§12)→ 判"输入不可比,存疑",维持冻结转人工,不下"算错"结论**——收敛风暴中的合法删除不会被系统性冻结误判。
  - **可裁决面分级(如实):** 域内拓扑(Router/Network-LSA、IS-IS L1/L2 域内)可裁决;含域间/外部路由(Type 3/4/5/7、虚链路、多拓扑、SR 扩展)→ 输出"部分可裁决"(仅裁决域内段)或"不可裁决转人工"。**"可裁决 LSDB 占比"为交付验收的实测项(§14/§15)**;真实网络含大量域间/外部路由,自动收敛承诺以该占比为条件。
  - **仲裁与退出:** 同快照下参考与主实现两次不一致 → 人工仲裁;"拒绝应用 + 重启计算"至多一次,复现即停自动、转人工——流程必收敛,不振荡。
- **风暴对象的计算视图暂缓(线缆视图/计算视图分离):** 对 D 路锁定的风暴对象(高频 purge/重生),**数据库、确认、CSNP/PSNP 对账完全按标准处理**(线缆视图零破坏);仅**计算视图**暂缓:缓存该对象最新状态,暂缓窗口结束统一应用,窗口内计算对该对象去抖。要点:
  - **暂缓时长分档过 §8.4 守护:** 过境依赖锥内对象 ≤10 秒(或联动);锥外 60 秒、上限 300 秒。合法撤销(含运维下线源端产生的)必然最终生效,陈旧度有界。
  - **泛洪节奏控制的上界:** 对风暴对象的泛洪合并突发,但放行间隔**不得低于协议标准重传节奏**——IS-IS 以 minimumLSPTransmissionInterval(5 秒)量级、OSPF 以 MinLSArrival/重传定时器(1–5 秒)量级为上界(§12),保证邻居的重传/请求机制不会与本机节奏形成持续失配。
  - **对账自洽:** CSNP/DD 中该对象通告**"最后已泛洪版本"**而非"最后入库版本"——邻居据对账认为本机持有的就是已向其泛洪的版本,不会为尚未放行的新版本反复发出请求;新版本在节奏放行时泛洪并同步推进对账通告(协议改造点,计入 §2.4/§14)。
  - 高序列号持续重生与 purge 风暴同等纳入;上限触发 + 人工强制放行开关。
- **自隔离(引流):** 自监控(崩溃频率/内存曲线/**过境流量与预估引流面的偏差**)判定"本机不健康"时,缺省档**仅告警不引流**;自动引流须满足佐证前置(算错裁决确认,或经认证人工),用 BGP GRACEFUL_SHUTDOWN / max-metric 平滑迁走流量,自动化的自隔离必须可自动撤销。
- **邻域并发保护(范围显式):** 网管侧观察到同窗(5 分钟)≥K(缺省 3)台同域设备触发**自动引流或自动冷启动** → 判"疑似诱导攻击或全域性缺陷",冻结**这两类动作**转人工。**冻结范围仅限自动引流与自动冷启动;本地非引流自愈(删除熔断冻结、计算视图暂缓、看门狗回退、骨架入库、净化)不受冻结**——真实全域故障下每台设备的本地止血不被并发保护拖死。判别输入:是否伴随相称域内事件(大量邻居 overload 通告/会话震荡/域级告警)、是否跨多个 defect_id;人工闸门配超时升级(T_esc,缺省 15 分钟无人响应 → 升级告警级别并允许预授权的保守动作)。
- **保护性冷启动(回放循环兜底,带护栏):** 触发条件 = **同一循环窗口(60 分钟)内第 3 次"回放/首算期复位"**,且让位窗口内的 B′ 扫描未能产出可用条目 → 本轮启动丢弃 checkpoint、跳过 GR 回放、按普通冷启动从邻居全量重学。护栏(此动作破坏性高——GR helper 失效、瞬时全表撤销——按与引流同级治理):
  - **频率上限与退避:** 每设备冷启动后 24 小时内不再自动冷启动;窗口内再次进入回放循环 → 停止该协议自动拉起(保持下线隔离态)+ 最高告警转人工——反复自动冷启动的重收敛风暴被封顶。
  - **邻域并发保护覆盖(见上):** 同域同窗已有设备冷启动 → 本机的自动冷启动转请求人工;设备失联网管时退化为本地退避规则。
  - **语义如实:** 冷启动确定性打断的是**回放臂**(毒物只存在于本机 checkpoint 时,一次冷启动根治);毒物在网络中仍有来源时,冷启动后转入**重收型**循环——但这把"无报文可查"的回放循环转换成"每轮都有接入证据"的重收循环,对照统计获得样本、入口拦截获得目标,循环的每一轮都在向"可拦截"收敛。
  - 纳入攻击面治理(§13.2);触发阈值与窗口入 §12。
- **状态机看门狗动作:** 卡死 → 采集上下文快照(含卡住对象溯源号)→ 尝试状态回退/重新初始化该状态机(非整进程重启);C2 即:升主订阅卡死 → 看门狗触发 → 回退订阅状态 → 按预设回退策略(如降级冷订阅)让倒换完成 → 告警 + 卡住对象溯源号交归因。

### 8.7 倒换前健康闸门

NSR/主备倒换的前提是"备板状态可信",否则倒换把毒扩散到备板、冗余同时失效:

- **闸门校验项:** 备板协议状态机可达性自检;**主备 RIB 摘要一致性**;关键状态机无 stuck 标记;免疫库版本一致;最近 N 分钟备板无绊线事件。
- **RIB 摘要机制(新建能力,工作量入 §14):** 每路由 64 位哈希、模 2^64 可交换加和(安装 +h、撤销 −h),序无关、O(1) 在线维护。**比对协议:** 主板在 NSR 同步通道插入**栅栏序号 b**;备板确认消费至 b 后,双方交换"截至 b 的摘要"——比对点由栅栏定义,在途更新不产生误报。不一致 → 重试下一栅栏(≤3 次)→ 仍不一致:**禁止自动倒换** + 最高告警 + 增量审计定位差异段;**手动强制倒换保留**(经认证 + 显式确认 + 记录),运维在"备板可能不健康"与"主板已不可用"间自行取舍,不被系统卡死。
- 失败处置:闸门不过 → 禁自动倒换 + 告警;主板已不可用的强制路径如上。

## 9. 免疫条目分发(全网受控免疫)

- **不自动扩散:** 本机学习的条目缺省只在本机生效。
- **中心化受控分发:** 设备上报(条目 + 证据档案)→ 网管人工审核 → 中心签名 → 按域灰度下发(观察态先行)→ 全量执行态。**接收端不重跑本地观察期**:该条目已在哨兵设备完成观察期(§12)与人工审核,网管灰度中的金丝雀批次(先在 1–4 小时短窗内以观察态校验环境差异)承担跨设备差异检验——这是灰度纪律的组成而非绕过。
- **审核门槛与证据档位联动:** E3 单人审核;E2 及以下双人;影响面预估超阈(§12)强制双人 + 分域灰度。
- **加速审批通道(压缩免疫窗口):** 崩溃类、档位 ≥E2 的条目走加速流程——签名、金丝雀、可回滚一个不减,只压时限。**端到端"哨兵发作 → 邻居执行态"的目标时延:哨兵观察期已满足时为小时级;哨兵观察期未满的紧急场景,可下发为观察态,或经双人紧急审批直接执行态并显式记录风险**——"小时级免疫"是有条件的目标,不是无条件承诺。
- **回滚指令**同通道下发,全网可撤;分发/生效/回滚全审计。

## 10. 典型案例端到端走查

### 10.1 C1(LSID=0 → 误删路由)

1. 稳态:该 Network-LSA 入库时,ls_id=0.0.0.0 命中缺省评分先验(保留地址域),异常分高 → 样本进隔离池;准入双条件(§2.5.5)使它**始终留在池中、永不刷成基线**(长窗中该值持续罕见,复评不回落)。若运营商启用了本地增强校验(保留域带处置),入口即净化,后续不发生——两条路径并存,缺省路径继续:
2. 数日后 SPF 计数达特殊值,触发误删 4.8 万路由。绊线(3) 触发:ΔRIB 巨大、时间轴上无相称撤销/事件 → **冻结**删除下发与撤销传播,最高告警。
3. 算错裁决:导出带版本戳快照 → 独立参考计算器(域内拓扑可裁决)确认"这批前缀不该删" → **本机算错**确证。
4. 归因:B 驻留扫描点亮该 LSA(长窗罕见 + 先验 + 首见年龄);因果邻近检查:它在被删前缀的依赖锥内。
5. 排除验证(受保护试算):冻结期、GR 保护下,排除该 LSA 重启计算——**两种结局如实分叉**:(i) 试算复现症状且排除后恢复 → 因果闭合,E3;(ii) 试算不复现(计数状态已变,无论排除与否结果都正常)→ 输出"无判别力",证据 = 值域异常 + 算错确证 = **E2**。两种结局下,当前恢复都已达成(重算结果正常即解冻应用)。
6. 出条目:`ls_id=0.0.0.0`(掩码变化字段),fault_path=compute,处置=隔离(计算侧);**一致性守护**:双向连通性校验证明该 LSA 本就不入正确计算(免联动证明 (a) 成立)→ 允许单独隔离,无需引流。E3 → 自动观察态,人工确认后执行态;E2 → 观察态 + 人工晋级(或第二次发作自动补齐因果证据)。
7. 复发 / 邻居设备:入口命中 → 隔离,不再进计算 → 误删不再发生;经网管审核分发全网。

### 10.2 C1-R(毒化 LSA 进入 checkpoint 的回放循环)

1. 已有执行态条目的设备:重启后条目**先于回放加载**(§2.5.3)→ 回放完成即驻留清理(§7.6):该 LSA 隔离出计算(守护证明 (a) 照旧成立)→ 循环打断,业务恢复。
2. 无条目的首台设备:崩溃 → 回放 → 首算再崩。循环检测(30 分钟 2 次)→ 循环模式:抬保真度、启用钉住区。
3. 下一周期:回放完成 → **让位窗口**(暂缓首次计算 ≤5 秒)→ B′ 回放态扫描(无写锁争用,窗口有保证)点亮该 LSA → 溯源标签反查摘要仓(在则补齐来源链,失引则如实降级)→ 出条目(观察态,值域异常成立)→ 本轮驻留隔离(过守护)→ 循环打断。
4. 兜底:协议不支持暂缓 / 扫描超时 / 回放期反复崩溃 → 60 分钟窗口内第 3 次回放期复位 → **保护性冷启动**(丢 checkpoint、跳过回放、全量重学)。回放臂确定性打断;该 LSA 若在网络中仍存活,重学后转为重收型——每一轮都产生接入证据,C 路对照与入口拦截由此接手;24 小时退避与邻域并发保护封顶重收敛风暴。

### 10.3 C3(IPsec 丢弃路径 UAF → 复位循环)

1. 首次崩溃:复位类型决定证据等级——软复位:保留 DDR 全量时间轴可用;**硬看门狗:取决于平台档位**(§5.1)——支持逐条 SRAM 镜像的平台,崩溃前最后一条接入的头指纹必然在 SRAM;仅支持周期镜像的平台,**如实声明该组合下真凶指纹大概率丢失**,归因依赖后续循环中的对照统计(重收型循环每轮都有新接入,SRAM 逐条档外的平台靠多轮 DDR 存活窗口或转人工)。
2. 循环模式:保真度抬高,该上送类样本进钉住区(或 SRAM 钉住区)。
3. C 路对照统计:目的为本机的 IPsec 上送类在每个崩溃前窗口出现率高、对照窗口(稳态基线快照)出现率低 → 提升度极高、入围;周期背景与每次不同的无辜报文被压掉。
4. 处置结果记录:该归一化类历史处置 **100% 策略丢弃**(record_verdict 证据),观测窗覆盖策略状态要求(§7.5)→ 处置等价性成立,E2 单臂出条目。
5. 条目:头部匹配条件 + 提前丢弃,policy_ref 绑定该丢弃策略版本;编译下压 CoPP/ACL → 报文不再上送 → 循环打断,零误杀(本就要丢);策略若变更,条目自动挂起复核。

### 10.4 C4(计时器快 100 倍 → purge 风暴)

1. 全网视角:各设备绊线(4) 高频对象统计锁定同一 LSP 的病理性重复(σ 高、无相称事件)→ 归本系统。
2. 处置(每台设备本地):**计算视图暂缓**——数据库/确认/CSNP 对账完全标准(线缆视图零破坏);该 LSP 的计算应用暂缓,窗口分档过守护:该 LSP 始发者若在本机过境依赖锥内 → T_hold ≤10 秒(与去抖同量级,只削震荡频率不制造长窗不一致);锥外 → 60 秒。泛洪合并突发但放行间隔不低于协议标准节奏(§8.6),CSNP 通告已泛洪版本——邻居对账不失配。
3. 计算风暴削峰、路由震荡遏制;**合法 purge(如运维下线)最终必然生效**(暂缓有界)。
4. 源头设备:自监控(始发速率远超基线)→ 告警;引流不自动(需佐证);运维修复内存跳变。
5. 不产出报文值条目——报文完全合法,正确结论就是行为处置 + 源头修复。

### 10.5 C2(EVPN IPv6 下一跳经 IPv4 会话 → 倒换卡死)

**主路径(能力校验,首台即防):**

1. 支持矩阵(register_caps 提交)声明:本机 EVPN 下一跳仅支持 IPv4 编码(经 IPv4 会话)。
2. 友商邻居经 IPv4 会话通告带 IPv6 下一跳的 EVPN 路由(RFC 7432/4760 合法,不经能力协商,无法在协商层拒绝)→ 入口能力校验命中矩阵外组合 → **定义化降级**:按撤销处理/不参与选路 + 告警"邻居 X 发送本机不支持的 EVPN IPv6 下一跳路由"。
3. 截断代码路径**根本不被触及**——不存在毒化下一跳,倒换正常。该目的地经此路径不可达的代价可见、可告警,待本机版本支持或对端调整后消除。

**纵深路径(支持矩阵存在遗漏时):**

4. 该路由流入订阅路径,倒换时截断 → 升主状态机卡死 → 绊线(2) 看门狗触发 → 回退订阅状态、按预设降级冷订阅 → **倒换完成,业务止血**(不依赖归因)。
5. 健康闸门:倒换前 RIB 栅栏摘要不一致/卡死标记 → 阻止把毒扩散到"仅存的好板"。
6. 归因:卡住对象溯源号 → 本板溯源摘要仓(NSR 随对象同步写入、免逐出位保护)→ 解析出该路由的归一化字段:**下一跳长度 16、EVPN、IPv4 会话——落在支持矩阵之外** → 归因结论"卡死溯源到一条能力范围外的输入",E3(确定性依据)。摘要若因极端容量压力被逐出 → 如实告警"溯源证据已老化",归因降级(止血与闸门不受影响)。
7. 产出**能力校验补充项**(kind=capability_supplement,处置=定义化降级)→ 人工确认并入支持矩阵 → 本机复发与全网设备此后在入口拦住。
8. 变体对照:某机首见一类**支持矩阵内**的合法路由(统计"新奇")→ 机械前置(§6.6)封顶观察态——能力校验拦的是"矩阵外",不是"没见过"。

## 11. 性能与资源预算

**成本口径按协议对象计**(一个 UPDATE 含 k 条 NLRI = k 个对象),逐项预算:

| 动作 | 每对象成本 | 说明 |
|---|---|---|
| 免疫库预筛 | 20–40ns | 预筛键 = (对象类型 ‖ 锚字段值),锚字段为条目条件中的等值项,编译期从条目集提取;两级哈希 O(1);区间/速率类条件进受控规模二级短表(数量受 §7.5 上限约束)。预筛表 ≤64KB 常驻缓存 |
| 规范 + 能力校验 | 10–30ns | 定长字段值域/矩阵查表,编译期展开 |
| 命中后全匹配 | 50–200ns | 仅命中预筛的少数对象 |
| 指纹(归一化关键字段) | 10–50ns | 类键哈希 + 变化字段哈希两段;**不哈希全报文** |
| 接入日志环写(小条目) | 30–60ns | 普通存储写 + release 头部;时戳读取 10–20ns 含在内 |
| 大样本写(全量,仅新奇/钉住) | +200–500ns | 绕过缓存写 + 存储栅栏(约 +30ns);受配额约束,非每对象 |
| 溯源标签 + 摘要晋级 | 10–30ns | 对象内 8B + 摘要仓槽写(仅创建驻留对象时) |
| 处置结果回填 | ~0 | 条目一次性发布时内联,无第二次环写 |
| 状态事件上报 | 30–60ns/条 | 批量聚合后摊薄 |
| **合计(典型路径)** | **≤300ns/对象** | 工作例:100 万路由全表收敛 30 秒内到达 ⇒ 钩子总耗时 ≈ 0.3 秒 ⇒ 单核占用 ≈1%,收敛时延回归预算 ≤2%(§12 验收线) |

- **验收实测项:** 多写者并发 + PAD 消费下的匹配延迟与环写吞吐;**绕过缓存写路径的撕裂率/半写误丢率**(正常运行应为零,崩溃时由 CRC 判弃);收敛时延回归 ≤2%。
- **硬件卸载覆盖面(如实):** 可下压 = 纯定长头字段条件(C3/LDM 类)与 CoPP 速率类;**BGP 条件一律不可下压**(TCP 字节流,NP 不做流重组);OSPF/IS-IS 仅浅层定长偏移可判者。卸载定位是"该子集零 CPU",不外推。
- 触发器稳态成本见 §4.4;归因/扫描/裁决只在触发后运行,受 PAD 资源上限与 worker 分片纪律约束。

## 13. 风险与规避

### 13.1 误报治理与防护动作的自我约束

- **分层误报口径(可测):** 条目误命中率——执行态前观察期须满足最小曝光(预期命中 ≥20 次);绊线误触发率——零事件时按曝光量给 95% 置信上界(≈3/曝光量),目标 <0.1 次/设备·年需 ≥30 设备·年曝光,达成前以实验室注入合法大规模变更的回归通过率替代并标注口径。
- **自动挂起/回滚判据(双条件):** 时间相关性(损害指标与该条目命中同窗)**且非预期性**——对引流/守护联动类,损害判定基准是**实际影响与证据档案预估引流面之差**:落在预估内 = 守护按设计生效,不是损害;超出预估依赖锥 → 告警人工。**带守护联动的条目不进入自动回滚集**(撤销引流可能把环路面放回来),只走"超预估告警 + 人工"通道。合法割接/TE 与条目命中偶然同窗不再构成误回滚。
- **防降级攻击:** 挂起/回滚前核查条目遥测——命中率高 + 命中样本异常分高 + 档案显示挂起将重开已知死循环 → 不自动挂起,升最高告警转人工;同一条目反复触发挂起/回滚 → 判"疑似被诱导"锁定人工。
- **逃生开关(分级):** 单条目挂起(运维直接)/ 协议级暂停(直通)/ 生效范围级 / 全局关闭。全局关闭:强认证 + 双人确认 + **自动到期(缺省 2 小时,可续)** + 关闭期条目仍以观察态记录(停执行不停取证)+ 全审计;恢复走审批。

### 13.2 攻击面汇总(对抗视角)

| 攻击 | 防线 |
|---|---|
| 构造报文诱导条目生成(污染免疫库) | 出闸门因果证据 + 人在环 + 观察期 + 影响面挂起线 |
| 诱导删除熔断做 DoS | 熔断=冻结不引流;引流佐证前置;邻域并发保护 |
| 诱导自隔离/守护联动做流量牵引 | 缺省仅告警;佐证前置;并发保护 K=3;非预期偏差回滚判据 |
| **诱导保护性冷启动制造重收敛风暴** | 循环窗口 + 24h 退避 + 邻域并发覆盖冷启动 + 再触发转人工 |
| 新奇洪泛冲刷黑匣子证据 | 捕获配额 + 钉住区 + 冲刷即绊线 |
| 基线投毒(慢速喂毒洗白) | 训练卫生四纪律(双窗/双条件准入/首见时间/缺省先验)+ 网管交叉校验 |
| 触发挂起/回滚做降级攻击 | §13.1 反诱导核查 + 人工锁定 |
| 篡改本地免疫库注入恶意条目 | HMAC/TPM + 双份 + 拒载 |
| **篡改双份致全库拒载 → 空库运行** | 如实承认 fail-open 决策:空库窗口重收型无入口拦截;自动网管重下发(分钟级恢复)+ 可选 fail-secure 档;攻击价值与恢复路径显式化 |
| 伪造网管分发通道 | 中心签名 + 通道认证 + 审计 |

### 13.3 处置引发路由不一致(专项)

单机偏离计算输入的全部处置统一过 §8.4 守护;守护的免环保证以"存在替代路径"为前提,割点场景由依赖锥挂起线导人工;守护联动并发上限防长期非过境;异厂商邻居永不同步隔离的现实是守护存在的根本理由——不宣称"邻居会配合",只保证"本机偏离时要么可证无害、要么显式退出过境"。

### 13.4 固有局限(如实列出)

1. 需要学习的条目,首个受害者中招(规范/能力两道门覆盖的除外);纯竞态提不出报文特征;支持范围内的合法输入 + 内部缺陷归自愈。
2. 学习条目存在逃逸空间(破坏性处置维持最小匹配条件的清醒代价;宽度联动 + 变体扩展 + 先验兜底缓解,不消除)。
3. 免疫窗口内哨兵仍向邻居泛洪毒化对象(§8.5 显式决策;加速审批压缩窗口,人工可升级拆邻接)。
4. 掉电首轮不可归因;**硬看门狗复位平台按 SRAM 档位分级——不支持逐条镜像的平台上,C3 类循环的真凶指纹大概率丢失**(§5.1/§10.3)。
5. 缓慢劣化类:触发可及,归因大概率低置信(真凶多已出窗);价值在预警与提前抬保真度。
6. 数据面硬件位错、伪装完全合法的拜占庭输入:范围外。
7. **保护性冷启动只打断回放臂**;网络源毒物使循环转为重收型,依赖入口拦截(需条目/能力项)承接——转换的收益是每轮循环开始产生可归因的接入证据。
8. **空库窗口**(双份完整性失败):重收型循环无入口拦截,恢复依赖网管重下发/人工;可选 fail-secure 档。
9. **一致性守护以"存在替代路径"为前提**,割点场景守护失效转人工;守护联动本身以引流(过境角色退出)为代价——把隐蔽环路风险显性化为可观测可回滚的运营代价。
10. 排除验证对"累计内部状态触发"的缺陷可能无判别力(输出如实标注,证据链退回 E2 + 人工);独立参考计算器的可裁决面限域内拓扑,含域间/外部路由的场景部分可裁决或转人工(占比实测,§14)。
11. 置信度数值在标定闭环建成前不用于决策(决策绑证据档位)。

## 15. 度量指标

- **检测轨:** 触发器覆盖率(测试床注入口径);绊线误触发率(§13.1 口径);MTTD。
- **归因轨:** top-3 命中率(注入口径);证据档位分布;排除验证"无判别力"率;**跨事件对照退化事件数**(对照不足降级 B/B′ 的次数);**独立参考计算器可裁决 LSDB 占比**(现网实测)。
- **免疫轨:** 条目产出时延;审核时延;命中数;误命中率(≥20 次曝光口径);**免疫窗口时长**(哨兵发作 → 邻居执行态);**免疫窗口内扩散事件数**。
- **自愈轨:** 异常事件 MTTR(有/无本系统对照);**首障止血时间**(设备级纯系统能力:发作 → 冻结/回退/暂缓生效);熔断误触发率;看门狗回退成功率;冷启动次数与转人工率;故障扩散设备数。
- **工程轨:** 热路径每对象成本实测;收敛时延回归(≤2%);**并发撕裂率/半写误丢率**;空库态时长;PAD 资源占用。
- **北极星(两个,分别归因):** 首障止血时间(设备级);全网受控免疫时间(网络级,含人工审核流程,单独归因,不与设备级混同)。

## 16. 依赖与复用的标准

- **BGP:** RFC 7606(治错阶梯,定义化降级与净化的语义基础)、RFC 4271/4760、**RFC 7432(EVPN;下一跳 v4/v6 按 MP_REACH_NLRI Next Hop 长度判别,不经能力协商——能力校验存在的直接依据)**、RFC 8950(IPv4 类 NLRI 的 IPv6 下一跳,注:不含 EVPN)、RFC 4724(GR)、RFC 8538(会话级降级语义)、RFC 7854(BMP,可选富化)、RFC 9552(BGP-LS;拓扑抽象导出,不承诺逐字节保真——不作训练源与裁决输入,仅旁路参考)。
- **OSPF:** RFC 2328(含 MinLSArrival——泛洪节奏上界依据)、RFC 8362;**RFC 6987(max-metric:仅置过境链路最大代价,割点仍会被用作过境——§8.4 语义注记的依据)**。
- **IS-IS:** ISO 10589(**overload bit:改变过境角色,不停止泛洪与接收;minimumLSPTransmissionInterval——节奏上界依据**)、RFC 5301-5311。
- **通用:** RFC 5880(BFD)、IEEE 802.1AX(LACP)——红线保护对象;RFC 7911(add-path)。
